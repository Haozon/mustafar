## 🏗️ 架构设计

### 类结构

```python
class LlamaDiffSparseKVAttention(nn.Module):
    """
    DiffSparseKV Attention 实现
    
    不继承 MUSTAFAR，独立实现
    """
    
    def __init__(self, config):
        # 实例变量存储
        self.window_state = None  # 窗口状态
        self.diff_sparse_thresholds = None  # 阈值
        self.thresholds_computed = False
        
        # 组件初始化
        self.importance_calculator = DiffKVImportanceCalculator()
        self.threshold_manager = GlobalThresholdManager(...)
        self.sparsity_applier = SparsityClassifierApplier(...)
        
        # GQA 配置检查
        if self.num_heads % self.num_key_value_heads != 0:
            raise ValueError(...)
        self.num_key_value_groups = self.num_heads // self.num_key_value_heads
```

**关键决策**：
- ✅ **不继承 MUSTAFAR**：独立实现，复制需要的函数
- ✅ **实例变量存储**：window_state 和 thresholds 存储在实例中
- ✅ **GQA 配置检查**：确保配置正确，避免运行时错误

### Window State 结构

```python
self.window_state = {
    # KV Cache 部分
    'compressed_keys': torch.Tensor or None,      # [B, H_kv, N_compressed, D]
    'compressed_values': torch.Tensor or None,    # [B, H_kv, N_compressed, D]
    'window_a_keys': torch.Tensor or None,        # [B, H_kv, <=W, D]
    'window_a_values': torch.Tensor or None,      # [B, H_kv, <=W, D]
    'window_a_size': int,                         # 实际大小 (0 到 W)
    'window_b_keys': torch.Tensor or None,        # [B, H_kv, <=W, D]
    'window_b_values': torch.Tensor or None,      # [B, H_kv, <=W, D]
    'window_b_size': int,                         # 实际大小 (0 到 W)
    
    # Attention 累积部分
    'accumulator': torch.Tensor,                  # [B, H_kv, W]
    'token_observation_count': torch.Tensor,      # [B, H_kv, W] - 关键！
}
```

**关键决策**：
- ✅ **H_kv 维度**：使用 num_key_value_heads，不是 num_heads
- ✅ **token_observation_count**：跟踪每个 token 的实际观察次数
- ✅ **动态大小**：window_a_size 和 window_b_size 可能 < W

## 🔄 Prefill 阶段

### 处理流程

```python
def _prefill_with_diff_sparse(self, hidden_states, attention_mask, position_ids, ...):
    """
    Prefill 阶段处理
    
    步骤：
    1. 计算 attention weights（不使用 Flash Attention）
    2. 计算 importance scores
    3. 计算并保存 per-layer thresholds
    4. 应用稀疏化
    5. 初始化双窗口
    """
```

### 两种情况

#### 情况 A: Prefill 长度 >= W

```python
if prefill_length >= W:
    # 稀疏化前 (prefill_length - W) 个 tokens
    prefix_keys = key_states[:, :, :-W, :]
    prefix_values = value_states[:, :, :-W, :]
    
    # 应用稀疏化
    compressed_keys, compressed_values = self.sparsity_applier.classify_and_apply_sparsity(
        prefix_importance, prefix_keys, prefix_values,
        thresholds=(threshold_high, threshold_low)
    )
    
    # 最后 W 个保持 dense，作为 Window A
    window_a_keys = key_states[:, :, -W:, :]
    window_a_values = value_states[:, :, -W:, :]
    window_a_size = W
```

#### 情况 B: Prefill 长度 < W

```python
else:
    # Prefill < W，全部作为 Window A，不稀疏化
    compressed_keys = None
    compressed_values = None
    window_a_keys = key_states
    window_a_values = value_states
    window_a_size = prefill_length
```

**关键决策**：
- ✅ **分情况处理**：根据 Prefill 长度选择不同策略
- ✅ **保留末尾 W tokens**：作为 Window A 的初始内容
- ✅ **Prefill < W 不稀疏化**：等待 Decode 阶段填充

### 阈值计算和保存

```python
# 计算 per-layer thresholds
threshold_high, threshold_low = self.threshold_manager.compute_per_layer_thresholds(
    importance_scores  # [B, H_kv, T]
)

# 保存到实例变量
self.diff_sparse_thresholds = (threshold_high, threshold_low)
self.thresholds_computed = True
```

**关键决策**：
- ✅ **Per-layer thresholds**：每层独立计算和保存
- ✅ **实例变量存储**：不通过 past_key_value 传递
- ✅ **Decode 复用**：Decode 阶段不重新计算

## 🔁 Decode 阶段

### 处理流程

```python
def _decode_with_diff_sparse(self, hidden_states, attention_mask, position_ids, ...):
    """
    Decode 阶段处理
    
    步骤：
    1. 检查 thresholds 是否已计算
    2. 生成新 token 的 key/value
    3. 调用 decode_step_with_dual_window
    4. 自动检查和触发压缩
    """
```

### Decode 单步处理

```python
def decode_step_with_dual_window(self, query_states, new_key, new_value, attention_mask):
    """
    单步 Decode 处理（支持 GQA）
    
    步骤：
    1. 添加新 token 到 Window A 或 Window B
    2. 构建完整 KV cache
    3. 计算 attention（需要 repeat_kv）
    4. 聚合 attention 到 KV head 维度（GQA）
    5. 累积对 Window A 的 attention
    6. 检查是否触发压缩
    """
```

### 两种情况

#### 情况 A: Window A 已满

```python
if window_a_size == W:
    # 添加到 Window B
    window_b.append(new_token)
    window_b_size += 1
    
    # 累积 attention
    accumulator[:, :, 0:W] += window_a_attention
    token_observation_count[:, :, 0:W] += 1
    
    # 检查压缩
    if window_b_size >= W:
        trigger_compression()
```

#### 情况 B: Window A 未满

```python
else:
    # 添加到 Window A
    window_a.append(new_token)
    window_a_size += 1
    
    # 累积 attention（范围动态）
    accumulator[:, :, 0:window_a_size] += window_a_attention
    token_observation_count[:, :, 0:window_a_size] += 1
    
    # 不触发压缩
```

**关键决策**：
- ✅ **动态累积**：无论 Window A 是否满，都累积 attention
- ✅ **token_observation_count**：跟踪每个 token 的实际观察次数
- ✅ **压缩条件**：Window A 满 AND Window B 满

### Attention Mask

```python
# Decode 阶段的 attention mask
# Query: [B, H_q, 1, D] - 新生成的 token
# Keys: [B, H_kv, T_total, D] - 所有历史 tokens

# 新 token 可以看到所有历史 tokens
# attention_mask 通常是 None 或全 0

if attention_mask is not None:
    attn_weights = attn_weights + attention_mask
    attn_weights = torch.max(attn_weights, torch.tensor(torch.finfo(attn_weights.dtype).min))
```

**关键决策**：
- ✅ **保留参数**：支持特殊情况
- ✅ **通常为 None**：Decode 阶段不需要 causal mask

### 压缩触发

```python
def _trigger_compression(self):
    """
    触发 Window A 的压缩并执行窗口滑动
    
    步骤：
    1. 归一化 attention（动态计算观察次数）
    2. 使用 Prefill 的 thresholds 分类
    3. 应用稀疏化
    4. 合并到已压缩区域
    5. 窗口滑动：Window B → Window A
    6. 重置 Window B 和累积器
    """
```

### 归一化公式（动态）

```python
for i in range(W):
    # 动态计算观察次数
    N_internal = W - 1 - i  # Window A 内部
    N_external = token_observation_count[:, :, i]  # 实际累积次数
    N_total = N_internal + N_external
    
    # 避免除零
    N_total = torch.clamp(N_total, min=1.0)
    
    # 归一化
    ᾱ[:, :, i] = Acc[:, :, i] / N_total
```

**关键决策**：
- ✅ **动态计算**：使用 token_observation_count
- ✅ **避免除零**：clamp 到最小值 1.0
- ✅ **准确性**：考虑 Window A 填充期间的不同观察次数

## 🔧 GQA 支持

### 配置检查

```python
def __init__(self, config):
    # 检查 GQA 配置
    if self.num_heads % self.num_key_value_heads != 0:
        raise ValueError(
            f"num_heads ({self.num_heads}) must be divisible by "
            f"num_key_value_heads ({self.num_key_value_heads})"
        )
    
    self.num_key_value_groups = self.num_heads // self.num_key_value_heads
```

### Repeat KV

```python
# 在计算 attention 前 repeat
attn_weights = torch.matmul(
    query_states,  # [B, H_q=32, 1, D]
    repeat_kv(full_keys, self.num_key_value_groups).transpose(2, 3)  # [B, H_q, T, D]
) / math.sqrt(self.head_dim)
```

### Attention 聚合（使用 Mean）

```python
# attn_weights: [B, H_q=32, 1, T_k]
# 需要聚合到 [B, H_kv=8, 1, T_k]

# Reshape 并聚合
attn_weights_grouped = attn_weights.view(
    B, H_kv, self.num_key_value_groups, 1, T_k
)  # [B, H_kv, group_size, 1, T_k]

# 使用 mean 聚合（保持值域 [0, 1]）
attn_weights_kv = attn_weights_grouped.mean(dim=2)  # [B, H_kv, 1, T_k]
```

**为什么使用 Mean**：
- ✅ 保持值域 [0, 1]，与 attention weights 的语义一致
- ✅ 更容易调试（值在合理范围内）
- ✅ 与单 head 的情况一致（num_groups=1 时，mean=sum）
- ✅ 在排序意义上与 sum 等价

### Accumulator 维度

```python
# 对于 GQA
B = batch_size
H_kv = num_key_value_heads  # 8, 不是 32
W = window_size

# Accumulator 维度
accumulator: [B, H_kv, W]  # [B, 8, 32]
token_observation_count: [B, H_kv, W]  # [B, 8, 32]
```

**关键决策**：
- ✅ **H_kv 维度**：与 KV cache 的维度匹配
- ✅ **Mean 聚合**：从 Query heads 聚合到 KV heads
- ✅ **配置检查**：确保 GQA 配置正确

## 🚨 边界情况处理

### 1. Window B 未满时生成结束

```python
状态：
- Window A: 32 tokens (满)
- Window B: 20 tokens (未满)
- 生成结束（遇到 EOS token）

处理：
- ❌ 不触发压缩
- Window A 和 Window B 保持 dense
- 累积的 attention 信息被丢弃
```

### 2. Window A 未满时生成结束

```python
状态：
- Window A: 25 tokens (未满)
- Window B: 空
- 生成结束（遇到 EOS token）

处理：
- ❌ 不触发压缩
- Window A 保持 dense
- 累积的 attention 信息被丢弃
```

### 3. Prefill 长度为 0

```python
if prefill_length == 0:
    raise ValueError("Prefill length cannot be 0")
```

### 4. Thresholds 未计算

```python
if not self.thresholds_computed or self.diff_sparse_thresholds is None:
    raise RuntimeError("Thresholds not computed. Must run prefill first.")
```

**设计理由**：
- ✅ **保持简单**：不强制压缩
- ✅ **短生成无损**：保持 dense，不会有质量损失
- ✅ **错误检查**：防止运行时错误

## 🛠️ 实现细节

### Flash Attention

```python
# Prefill 阶段：可以使用 Flash Attention（如果不需要 attention weights）
# Decode 阶段：禁用 Flash Attention（需要 attention weights 进行累积）

def _prefill_with_diff_sparse(self, ...):
    # 不使用 Flash Attention，需要 attention weights
    attn_weights = torch.matmul(query_states, key_states.transpose(2, 3)) / math.sqrt(self.head_dim)
    attn_weights = F.softmax(attn_weights, dim=-1, dtype=torch.float32).to(query_states.dtype)
    
def _decode_with_diff_sparse(self, ...):
    # 不使用 Flash Attention，需要 attention weights
    attn_weights = torch.matmul(query_states, keys.transpose(2, 3)) / math.sqrt(self.head_dim)
    attn_weights = F.softmax(attn_weights, dim=-1, dtype=torch.float32).to(query_states.dtype)
```

**关键决策**：
- ✅ **Decode 禁用**：需要 attention weights 进行累积
- ✅ **优先正确性**：性能优化后续考虑

### 数据类型

```python
# Accumulator 使用 float32
accumulator = torch.zeros(B, H_kv, W, device=device, dtype=torch.float32)
token_observation_count = torch.zeros(B, H_kv, W, device=device, dtype=torch.float32)
```

**关键决策**：
- ✅ **float32 足够**：累积 32-64 次，精度足够
- ✅ **不需要 float64**：避免额外开销
