## 完整方案设计

### 核心参数配置

```
稀疏度等级系统：
Level 0: 0%稀疏  （完全保留）
Level 1: 50%稀疏 （保留50%的features）
Level 2: 70%稀疏 （保留30%的features）
Level 3: 90%稀疏 （保留10%的features）
Level 4: 100%稀疏（完全剪枝）

目标分布：
[5%, 15%, 30%, 30%, 20%]

预期平均稀疏度：
0.05×0% + 0.15×50% + 0.30×70% + 0.30×90% + 0.20×100%
= 0 + 7.5% + 21% + 27% + 20%
= 75.5%

双窗口配置：
Window B (保护窗口): 256 tokens, Level 0强制
Window A (观察窗口): 256 tokens, 等待稀疏化

阈值数量：4个
threshold_h:  Level 0 vs Level 1 的边界
threshold_mh: Level 1 vs Level 2 的边界
threshold_m:  Level 2 vs Level 3 的边界
threshold_l:  Level 3 vs Level 4 的边界
```

### Prefill阶段算法

```
输入：Prompt tokens

Step 1: 前向传播获取attention
outputs = model(prompt, output_attentions=True)
attention_weights = outputs.attentions  # List of [B, H, T, T]

Step 2: 计算所有tokens的importance
all_importance = []
for layer_idx, attn in enumerate(attention_weights):
    importance = compute_diffkv_importance(attn)  # [B, H, T]
    all_importance.append(importance)

# 展平所有层的importance
flattened_importance = concat_and_flatten(all_importance)
# 形状: [B * H * num_layers * T]

Step 3: 计算全局阈值（基于目标分布）
sorted_imp = sort(flattened_importance, descending=True)
n = len(sorted_imp)

# 计算累积分位点
cumsum = cumsum([0.05, 0.15, 0.30, 0.30, 0.20])
# = [0.05, 0.20, 0.50, 0.80, 1.00]

# 提取阈值
threshold_h = sorted_imp[int(0.05 * n)]   # 前5%之后
threshold_mh = sorted_imp[int(0.20 * n)]  # 前20%之后
threshold_m = sorted_imp[int(0.50 * n)]   # 前50%之后
threshold_l = sorted_imp[int(0.80 * n)]   # 前80%之后

保存：thresholds = [threshold_h, threshold_mh, threshold_m, threshold_l]

Step 4: 对每一层、每个token分配等级
for layer_idx in range(num_layers):
    importance = all_importance[layer_idx]  # [B, H, T]
    
    for b, h, t in iterate(B, H, T):
        imp = importance[b, h, t]
        
        if imp >= threshold_h:
            level[layer, b, h, t] = 0
        elif imp >= threshold_mh:
            level[layer, b, h, t] = 1
        elif imp >= threshold_m:
            level[layer, b, h, t] = 2
        elif imp >= threshold_l:
            level[layer, b, h, t] = 3
        else:
            level[layer, b, h, t] = 4

Step 5: 应用magnitude pruning
for each token:
    level = assigned_level[token]
    sparsity = SPARSITY_MAP[level]  # [0%, 50%, 70%, 90%, 100%]
    
    # 对K和V应用相同的稀疏度
    token.K = magnitude_topk(token.K, keep_ratio=1-sparsity)
    token.V = magnitude_topk(token.V, keep_ratio=1-sparsity)

输出：
- 稀疏化的prompt KV cache
- 保存的thresholds用于decoding
```

### Decoding阶段算法

```
初始化：
- Window_A = []  # 256 tokens
- Window_B = []  # 256 tokens
- thresholds = [threshold_h, threshold_mh, threshold_m, threshold_l]
  （从prefill加载）

主循环：每生成一个新token

Step 1: 新token加入Window B
new_token = generate_next_token()
Window_B.append(new_token)
# Window B中的tokens保持Level 0（完全保留，不稀疏化）

Step 2: 检查Window B是否满
if len(Window_B) >= 256:
    # 触发Window A的稀疏化
    compress_window_A()
    
    # 窗口滑动
    Window_A = Window_B
    Window_B = []

continue generation...

---

compress_window_A()函数：

如果Window A为空（第一次），直接返回

Step 1: 前向传播获取attention（只需要最后一个query）
# 实际上在生成过程中，attention已经计算了
# 我们需要收集Window A中tokens的attention history

# 关键：Window A的tokens在过去256步（Window B期间）
# 已经被256个新queries attend过了
# 我们需要累积这些attention来计算importance

accumulated_attention = collect_attention_for_window_A()
# 这个需要在生成过程中维护

Step 2: 计算Window A中tokens的importance
importance = compute_diffkv_importance(accumulated_attention)
# [B, H, 256] - Window A有256个tokens

Step 3: 对每一层，使用全局阈值分配等级
for layer_idx in range(num_layers):
    layer_importance = importance[layer_idx]  # [B, H, 256]
    
    for b, h, t in iterate(B, H, 256):
        imp = layer_importance[b, h, t]
        
        if imp >= threshold_h:
            level[layer, b, h, t] = 0
        elif imp >= threshold_mh:
            level[layer, b, h, t] = 1
        elif imp >= threshold_m:
            level[layer, b, h, t] = 2
        elif imp >= threshold_l:
            level[layer, b, h, t] = 3
        else:
            level[layer, b, h, t] = 4

Step 4: 应用magnitude pruning
for each token in Window_A:
    level = assigned_level[token]
    sparsity = SPARSITY_MAP[level]
    
    token.K = magnitude_topk(token.K, keep_ratio=1-sparsity)
    token.V = magnitude_topk(token.V, keep_ratio=1-sparsity)
```

### 关键技术细节

#### 细节1：Attention收集机制

```
问题：如何为Window A的tokens收集attention？

方案A：在线累积（推荐）
在generation过程中：
- 每一步生成时，新query对所有keys的attention已计算
- 维护一个attention accumulator：
  attention_acc[layer][token_id] += current_attention
- 256步后，Window A的tokens的attention_acc就是完整的

伪代码：
# 初始化
attention_accumulator = {}  # {layer_id: {token_id: sum_attention}}

# 每一步generation
def generation_step():
    outputs = model.generate_one_token(output_attentions=True)
    
    # 收集attention
    for layer_idx, attn in enumerate(outputs.attentions):
        # attn: [B, H, 1, L]  (single query对所有keys)
        for token_id in Window_A_token_ids:
            attention_accumulator[layer_idx][token_id] += attn[:, :, 0, token_id]
    
    # 256步后，用accumulator计算importance

方案B：批量计算（备选）
Window B满时，重新过一遍Window A：
- 把Window A的256个tokens作为keys
- Window B的256个tokens作为queries
- 计算一次完整的attention
- 但这需要额外的前向传播（开销大）

推荐方案A（在线累积）
```

#### 细节2：Per-layer还是全局阈值

```
你的方案：全局阈值

具体实现：
- 所有层的importance展平到一起
- 计算一组全局阈值
- 所有层使用相同的阈值判断

优势：
✓ 简单
✓ 跨层一致
✓ 容易理解和调试

问题：
⚠️ 不同层的importance scale可能不同
- 浅层的importance可能普遍较高
- 深层的importance可能普遍较低
- 用全局阈值可能导致浅层过度保护，深层过度稀疏

改进（可选）：
Per-layer阈值：
for each layer:
    layer_importance = importance[layer]
    sorted_imp = sort(layer_importance)
    
    # 每层独立计算阈值
    threshold_h[layer] = sorted_imp[int(0.05 * n)]
    threshold_mh[layer] = sorted_imp[int(0.20 * n)]
    ...

建议：
先用全局阈值（简单）
如果效果不好，尝试per-layer
```

#### 细节3：Importance的per-head还是平均

```
你的importance计算返回：[B, H, T]
保留了head维度

选择1：Per-head独立处理
- 每个head独立计算阈值和分配等级
- 同一个token在不同head可能有不同稀疏度

选择2：跨head平均
importance_avg = importance.mean(dim=1)  # [B, T]
- 所有heads对该token使用相同的稀疏度

建议：选择2（跨head平均）
理由：
1. 简单，KV cache实现更容易
2. 同一个token的K/V向量在所有head共享
3. 没必要per-head不同（GQA中多个query head共享一个KV head）

如果要per-head：
只在MHA（非GQA）且想要极致优化时考虑
```

### 完整的数据流

```
=== Prefill ===
Input: "Solve the equation: 3x + 5 = 20"
         ↓
    Forward pass
         ↓
    Attention weights [L, B, H, T, T]
         ↓
    compute_diffkv_importance()
         ↓
    Importance scores [L, B, H, T]
         ↓
    Flatten & sort
         ↓
    Compute thresholds [threshold_h, threshold_mh, threshold_m, threshold_l]
    例如: [2.5, 1.8, 1.2, 0.7]
         ↓
    Assign levels for each token
         ↓
    Apply magnitude pruning
         ↓
    Sparse KV cache (stored)

=== Decoding ===
Generate: "First,"
         ↓
    Add to Window B (Level 0, no sparsity)
         ↓
Generate: "subtract"
         ↓
    Add to Window B
         ↓
... (repeat 254 more tokens) ...
         ↓
Generate: "Therefore,"
         ↓
    Window B full (256 tokens)
    Trigger: Compress Window A
         ↓
    Collect attention for Window A tokens
    (accumulated over past 256 steps)
         ↓
    compute_diffkv_importance()
    Importance for 256 tokens in Window A
         ↓
    Use prefill thresholds [2.5, 1.8, 1.2, 0.7]
    Assign levels
         ↓
    Apply magnitude pruning to Window A
         ↓
    Window A → compressed (in-place)
    Window B → becomes new Window A
    Create empty Window B
         ↓
Continue generation...
```

## 潜在问题和应对

### 问题1：100%稀疏太激进

```
症状：
Level 4的20%tokens完全丢失，导致性能大幅下降

解决方案：
1. 降低Level 4比例
   [5%, 15%, 30%, 40%, 10%] → 只丢10%
   
2. 去掉Level 4
   改用[0%, 50%, 70%, 90%]
   分布[10%, 20%, 35%, 35%]
   
3. Level 4改成95%而不是100%
   仍保留5%的信息
```

### 问题2：阈值泛化性差

```
症状：
Decoding的importance分布严重偏离prefill
导致实际分布失控

解决方案：
1. 使用per-window排序
   放弃固定阈值，每个Window A内部排序
   
2. 使用层级阈值
   浅层和深层用不同阈值
   
3. 只在prefill用阈值
   Decoding用固定策略（所有generated tokens用Level 2）
```

### 问题3：收益不明显

```
症状：
相比uniform只提升<1%

解决方案：
1. 加入K/V differentiation
   这是下一步优化方向
   
2. 调整级别和分布
   尝试不同的sparsity配置
   
3. 分析哪些类型的tokens被错误分类
   改进importance计算
```

## 总结

你的简化后方案：

**✅ 核心设计**
- Fixed prefill阈值（不用EMA）
- 统一K/V稀疏度（不differentiation）
- 5级稀疏：[0%, 50%, 70%, 90%, 100%]
- 双窗口：256+256
- 目标分布：[5%, 15%, 30%, 30%, 20%]

**✅ 主要优势**
- 简洁清晰，容易实现
- 建立在验证过的技术上（Mustafar）
- 归一化importance设计正确
- 有明确的evaluation plan

**⚠️ 主要风险**
- 阈值泛化性（可能需要per-window排序）
- 100%稀疏可能太激进（可以调整）
