# 实现总结：聚合 Importance + Token Eviction

## 🎯 实现的功能

我们实现了两个关键改进：

1. **聚合 Importance Scores**：所有 heads 使用相同的 token 选择
2. **Token Eviction**：Level 2 (100% 稀疏) 的 tokens 使用丢弃而不是置零

---

## 📝 修改的文件

### 1. `DiffSparseKV/llama_diff_sparse_kv.py`

**修改位置：** `_prefill_with_diff_sparse` 方法，第 723-735 行

**修改内容：**

```python
# 原来：每个 head 独立计算 importance 和阈值
importance_scores = self.importance_calculator.compute_diffkv_importance(...)  # [B, H_kv, seq_len]
threshold_high, threshold_low = self.threshold_manager.compute_per_layer_thresholds(importance_scores)

# 现在：聚合 importance，所有 heads 使用相同的分类
importance_scores_per_head = self.importance_calculator.compute_diffkv_importance(...)  # [B, H_kv, seq_len]
importance_scores_aggregated = importance_scores_per_head.mean(dim=1)  # [B, seq_len] - 聚合
importance_scores = importance_scores_aggregated.unsqueeze(1).expand_as(importance_scores_per_head)  # 扩展回去
threshold_high, threshold_low = self.threshold_manager.compute_per_layer_thresholds(
    importance_scores_aggregated.unsqueeze(1)  # [B, 1, seq_len]
)
```

**效果：**
- ✅ 所有 heads 对每个 token 有相同的分类
- ✅ 可以精确控制压缩率（35% 就是 35%）
- ✅ 简化了后续的 token eviction 实现

#### 修改 A: `classify_and_apply_sparsity` 方法

**修改位置：** 第 115-130 行

**修改内容：**

```python
# 原来：总是使用 magnitude pruning
pruned_keys, pruned_values = self._apply_magnitude_pruning(...)

```

#### 修改 B: 新增 `_apply_token_eviction` 方法

**位置：** 第 256-320 行

**功能：**

```python
def _apply_token_eviction(self, key_states, value_states, sparsity_levels):
    """
    丢弃 Level 2 的 tokens，而不是置零
    
    步骤：
    1. 找到 Level 2 的 tokens (sparsity_levels == 2)
    2. 创建 keep_mask (保留 Level 0 和 Level 1)
    3. 使用 gather 只保留需要的 tokens
    4. 对 Level 1 的 tokens 应用维度级稀疏化（如果有）
    
    返回：
    - evicted_keys: [B, H, T_kept, D] - T_kept < T
    - evicted_values: [B, H, T_kept, D]
    """
```

**关键代码：**

```python
# 找到要保留的 tokens
keep_mask = (level_mask_per_token != 2)  # [T]
keep_indices = torch.where(keep_mask)[0]

# 丢弃 Level 2 的 tokens
evicted_keys = key_states[:, :, keep_indices, :]  # [B, H, T_kept, D]
evicted_values = value_states[:, :, keep_indices, :]

# 对 Level 1 应用维度级稀疏化
if num_level_1 > 0:
    self._apply_topk_pruning(evicted_keys, evicted_values, level_1_mask, ...)
```

## 🔍 工作流程

### Prefill 阶段

```
1. 计算 per-head importance scores
   ↓
2. 聚合 importance scores (mean across heads)
   ↓
3. 基于聚合的 importance 计算阈值
   ↓
4. 分类 tokens 到 3 个 levels (所有 heads 相同)
   ↓
5. 检查 Level 2 的稀疏度
   ├─ 如果 = 100%: 使用 token eviction
   │   ├─ 丢弃 Level 2 的 tokens
   │   └─ 对 Level 1 应用维度级稀疏化
   └─ 如果 < 100%: 使用 magnitude pruning
       └─ 对每个 level 应用维度级稀疏化
```

### 当前配置的行为

```python
TARGET_DIST = "0.65,0.0,0.35"
SPARSITY_LEVELS = "0.0,0.7,1.0"

# 分类结果：
# - 65% tokens → Level 0 (dense, 完全保留)
# - 0% tokens → Level 1 (70% 维度稀疏, 但没有 tokens)
# - 35% tokens → Level 2 (100% 稀疏, 丢弃)

# Token eviction:
# - 输入: [B, H, 1000, 128]
# - 丢弃 350 个 tokens
# - 输出: [B, H, 650, 128]

# 效果：
# - 内存减少 35%
# - 计算减少 35%
# - Softmax 只在 650 个 tokens 上归一化
# - 没有权重浪费
```

## 📊 预期效果

### 与之前（置零）的对比

| 指标 | 置零方式（旧） | Token Eviction（新） |
|------|--------------|-------------------|
| 形状 | [B, H, 1000, 128] | [B, H, 650, 128] |
| 内存 | 不变 | 减少 35% |
| 计算 | 不变 | 减少 35% |
| Softmax 分母 | 1000 项 | 650 项 |
| 权重浪费 | 35% | 0% |
| 信息保留 | 65% | 100% (of kept tokens) |

### 预期模型效果提升

**之前的问题：**
- 置零方式导致 35% 的 attention 权重被浪费
- 有效 tokens 的权重被稀释到只有 65%
- 多层累积后信息损失严重
- 模型分数很低（3.83, 8.92, 14.85）

**现在的改进：**
- Token eviction 避免了权重浪费
- 有效 tokens 得到 100% 的 attention 权重
- 信息不被稀释
- **预期模型分数应该显著提升**
