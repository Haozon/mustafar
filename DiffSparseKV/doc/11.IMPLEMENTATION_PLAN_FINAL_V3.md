## 🎯 双窗口机制详解

### 两个逻辑窗口

1. **Window A（可压缩窗口）**
   - 大小：最多 W tokens（可能 < W）
   - 作用：等待压缩的 tokens
   - 状态：在 Window B 填充期间被持续观察
   - 压缩时机：当 Window B 满时（W tokens）

2. **Window B（保护窗口）**
   - 大小：最多 W tokens（可能 < W）
   - 作用：新生成的 tokens，保持 dense
   - 状态：为 Window A 提供后续观察样本
   - 重置时机：满后滑动到 Window A

### Window State 存储

**存储方式**：使用实例变量（不通过 past_key_value 传递）

```python
class LlamaDiffSparseKVAttention:
```

### 与 MUSTAFAR 的对比

#### MUSTAFAR 机制（residual_length = 32）

```
结构：
┌─────────────────────────────────────────────────────────────┐
│ Compressed Cache │ Token[33] │ Residual Window (32 tokens) │
└─────────────────────────────────────────────────────────────┘
                    ↑ 每次 prune 这 1 个 token

每个 decode step:
1. 新 token 追加到末尾
2. 如果长度 > 32:
   - Prune 第 33 个 token（移出 residual window 的）
   - 保持最后 32 tokens dense
3. 频率：每个 token 都 prune 一次

特点：
- 滑动窗口，每次移动 1 个 token
- 每次只 prune 1 个 token
- 没有 attention 累积
```

#### DiffSparseKV 双窗口机制（W = 32）

```
结构：
┌─────────────────────────────────────────────────────────────┐
│ Compressed Cache │ Window A (32) │ Window B (0-32)         │
└─────────────────────────────────────────────────────────────┘
                    ↑ 批量压缩      ↑ 逐步填充

每 32 个 decode steps:
1. Window B 从 0 填充到 32
2. 累积对 Window A 的 attention
3. Window B 满时:
   - 批量压缩整个 Window A（32 tokens）
   - Window B → Window A
   - Window B 重置为空
4. 频率：每 32 tokens 压缩一次

特点：
- 批量窗口，每次移动 32 tokens
- 每次批量 prune 32 tokens
- 有 attention 累积机制
```

**关键区别**：
| 特性 | MUSTAFAR | DiffSparseKV |
|------|----------|--------------|
| 压缩频率 | 每 1 token | 每 32 tokens |
| 单次压缩量 | 1 token | 32 tokens |
| Attention 累积 | 无 | 有（32 steps）|
| 窗口机制 | 单窗口滑动 | 双窗口批量 |

## 🔄 完整工作流程

### 阶段 1: Prefill 结束后的初始化

#### 情况 A: Prefill 长度 >= W

```
Prefill 输出（例如 100 tokens）：
┌─────────────────────────────────────────────────────────────┐
│ Prefill Cache (前 68 tokens, 已压缩) │ 最后 W=32 tokens   │
└─────────────────────────────────────────────────────────────┘

初始化双窗口：
┌─────────────────────────────────────────────────────────────┐
│ Compressed (68) │ Window A (32) │ Window B (空)            │
└─────────────────────────────────────────────────────────────┘
                   ↑ Prefill 末尾   ↑ 等待新 tokens
                     
状态：
- compressed_cache: Prefill 的前 68 tokens（已压缩）
- window_a: Prefill 的最后 32 tokens（dense）
- window_a_size: 32
- window_b: 空
- window_b_size: 0
- accumulator: 初始化为 0 [B, H, W]
- token_observation_count: 初始化为 0 [B, H, W]
```

#### 情况 B: Prefill 长度 < W

```
Prefill 输出（例如 20 tokens）：
┌─────────────────────────────────────────────────────────────┐
│ Window A (20 tokens, 未满)                                  │
└─────────────────────────────────────────────────────────────┘

初始化双窗口：
┌─────────────────────────────────────────────────────────────┐
│ Compressed (空) │ Window A (20) │ Window B (空)            │
└─────────────────────────────────────────────────────────────┘
                   ↑ Prefill 全部   ↑ 等待新 tokens
                     
状态：
- compressed_cache: 空
- window_a: Prefill 的全部 20 tokens（dense）
- window_a_size: 20
- window_b: 空
- window_b_size: 0
- accumulator: 初始化为 0 [B, H, W]
- token_observation_count: 初始化为 0 [B, H, W]

注意：需要先填充 Window A 到 32 tokens，才开始填充 Window B
```

### 阶段 2: Decode 循环

#### 情况 A: Window A 已满（正常情况）

```
Step 1:
┌─────────────────────────────────────────────────────────────┐
│ Compressed │ Window A (32) │ Window B (1)                  │
└─────────────────────────────────────────────────────────────┘
                              ↑ 新 token

操作：
1. 生成新 token → 追加到 Window B
2. 计算 attention: α_1 = softmax(q_1 * K_full^T / sqrt(d))
3. 提取对 Window A 的 attention: α_1[window_a_range]
4. 累积: Acc[:, :, 0:32] += α_1[window_a_range]
5. token_observation_count[:, :, 0:32] += 1

Step 2-31:
- 重复上述过程
- Window B 逐渐填充: 2, 3, ..., 31 tokens
- 持续累积对 Window A 的 attention

Step 32:
┌─────────────────────────────────────────────────────────────┐
│ Compressed │ Window A (32) │ Window B (32) ← 满！          │
└─────────────────────────────────────────────────────────────┘

操作：
1. 生成新 token → Window B 满（32 tokens）
2. 计算 attention 并累积（最后一次）
3. ✅ 触发压缩事件！
```

#### 情况 B: Window A 未满（Prefill < W）

```
初始状态：
- Window A: 20 tokens (未满)
- Window B: 空

Step 1-12 (填充 Window A):
┌─────────────────────────────────────────────────────────────┐
│ Compressed (空) │ Window A (21-32) │ Window B (空)         │
└─────────────────────────────────────────────────────────────┘

操作：
1. 生成新 token → 追加到 Window A
2. 计算 attention
3. 提取对 Window A 的 attention
4. 累积: Acc[:, :, 0:window_a_size] += attention
5. token_observation_count[:, :, 0:window_a_size] += 1
6. window_a_size += 1
7. ❌ 不触发压缩（Window A 未满）

Step 13 (Window A 满了):
┌─────────────────────────────────────────────────────────────┐
│ Compressed (空) │ Window A (32) │ Window B (1)             │
└─────────────────────────────────────────────────────────────┘

操作：
1. 生成新 token → 开始填充 Window B
2. 继续累积 attention
3. ❌ 不触发压缩（Window B 未满）

Step 14-44:
- 继续填充 Window B
- 持续累积 attention

Step 44 (Window B 满了):
- ✅ 触发压缩事件！
```

### 阶段 3: 压缩事件（Window B 满时）

```
压缩前：
┌─────────────────────────────────────────────────────────────┐
│ Compressed │ Window A (32) │ Window B (32)                 │
└─────────────────────────────────────────────────────────────┘
              ↑ 要压缩

步骤：
1. 归一化 attention:
   for i in range(window_a_size):
       N_internal = window_a_size - 1 - i  # Window A 内部因果 attention
       N_external = token_observation_count[:, :, i]  # 实际累积次数
       N_total = N_internal + N_external
       ᾱ[:, :, i] = Acc[:, :, i] / N_total
       
2. 分类 tokens:
   使用 Prefill 的 (τ_high, τ_low) 分类：
   Level(i) = {
     0,  if ᾱ^(i) >= τ_high     (保留全部)
     1,  if τ_low <= ᾱ^(i) < τ_high  (保留 30%)
     2,  if ᾱ^(i) < τ_low       (完全删除)
   }
   
3. 稀疏化 Window A:
   compressed_window_a = apply_sparsity(window_a, levels)
   
4. 合并到已压缩区域:
   compressed_cache = concat(compressed_cache, compressed_window_a)

压缩后（窗口滑动）：
┌─────────────────────────────────────────────────────────────┐
│ Compressed (增加了 32) │ Window A (32) │ Window B (空)     │
└─────────────────────────────────────────────────────────────┘
                          ↑ 原 Window B   ↑ 重置

步骤：
5. 窗口滑动:
   window_a = window_b  (Window B → Window A)
   window_a_size = window_b_size
   window_b = None      (Window B 重置为空)
   window_b_size = 0
   
6. 重置累积器:
   accumulator.zero_()
   token_observation_count.zero_()
```

### 阶段 4: 生成提前结束的处理

#### 情况 A: Window B 未满时结束

```
状态：
- Window A: 32 tokens (满)
- Window B: 20 tokens (未满)
- 生成结束（遇到 EOS token）

处理：
- ❌ 不触发压缩
- Window A 和 Window B 保持 dense
- 累积的 attention 信息被丢弃
```

#### 情况 B: Window A 未满时结束

```
状态：
- Window A: 25 tokens (未满)
- Window B: 空
- 生成结束（遇到 EOS token）

处理：
- ❌ 不触发压缩
- Window A 保持 dense
- 累积的 attention 信息被丢弃
```

**设计理由**：
- 保持算法简单
- 短生成任务保持 dense，不会有质量损失
- LongBench 中大部分任务生成长度足够触发压缩

## 📐 关键公式

### 1. 观察次数计算（动态）

对于 Window A 中第 i 个 token（i ∈ {0, 1, ..., window_a_size-1}）：

```
N_total^(i) = N_internal^(i) + N_external^(i)

其中：
- N_internal^(i) = window_a_size - 1 - i  (Window A 内部因果 attention)
  * token i 能被 token i+1, i+2, ..., window_a_size-1 关注
  * 共 window_a_size - 1 - i 次

- N_external^(i) = token_observation_count[:, :, i]  (Window B 的实际累积次数)
  * 通过 token_observation_count 跟踪每个 token 的实际观察次数
  * 在 Window A 填充期间，不同 token 的观察次数可能不同

因此：
N_total^(i) = (window_a_size - 1 - i) + token_observation_count[:, :, i]
```

**示例（Prefill = 20, Window A 填充到 32）**：
```
压缩时（Window B 满，window_a_size = 32）:

Token[0] (Prefill 的第一个):
- N_internal = 32 - 1 - 0 = 31
- N_external = 44 (Step 1-44 都累积了)
- N_total = 75

Token[20] (Prefill 的最后一个):
- N_internal = 32 - 1 - 20 = 11
- N_external = 44 (Step 1-44 都累积了)
- N_total = 55

Token[21] (Decode 第一个):
- N_internal = 32 - 1 - 21 = 10
- N_external = 43 (Step 2-44 累积了，Step 1 没有)
- N_total = 53

Token[31] (Window A 最后一个):
- N_internal = 32 - 1 - 31 = 0
- N_external = 32 (Step 13-44 累积了)
- N_total = 32
```

### 2. 归一化重要性分数

```python
# 累积器: Acc[b, h, i] 存储 token i 被观察的总 attention
# Token 观察计数: token_observation_count[b, h, i] 存储 token i 的外部观察次数
# 归一化:
for i in range(window_a_size):
    N_internal = window_a_size - 1 - i
    N_external = token_observation_count[b, h, i]
    N_total = N_internal + N_external
    ᾱ[b, h, i] = Acc[b, h, i] / N_total
```

**物理意义**：ᾱ^(i) 表示 token i 在所有有效观察中的平均 attention 概率

### 3. Token 分类

```python
# 使用 Prefill 阶段的 thresholds
for i in range(window_a_size):
    if ᾱ[i] >= τ_high:
        level[i] = 0  # 保留全部
    elif ᾱ[i] >= τ_low:
        level[i] = 1  # 保留 30%
    else:
        level[i] = 2  # 完全删除
```

## 🔧 GQA (Grouped Query Attention) 支持

### GQA 基础

```python
# Llama 3 的配置
num_heads = 32  # Query heads
num_key_value_heads = 8  # KV heads
num_key_value_groups = 4  # 每个 KV head 对应 4 个 Query heads
```

### GQA 配置检查

```python
def __init__(self, config):
    # 检查 GQA 配置
    if self.num_heads % self.num_key_value_heads != 0:
        raise ValueError(
            f"num_heads ({self.num_heads}) must be divisible by "
            f"num_key_value_heads ({self.num_key_value_heads})"
        )
    
    self.num_key_value_groups = self.num_heads // self.num_key_value_heads
```

### Attention 聚合（使用 Mean）

```python
# attn_weights: [B, H_q=32, 1, T_k]
# 需要聚合到 [B, H_kv=8, 1, T_k]

# Reshape 并聚合
attn_weights_grouped = attn_weights.view(
    B, H_kv, num_key_value_groups, 1, T_k
)  # [B, H_kv, group_size, 1, T_k]

# 使用 mean 聚合（保持值域 [0, 1]）
attn_weights_kv = attn_weights_grouped.mean(dim=2)  # [B, H_kv, 1, T_k]
```

**为什么使用 Mean**：
- 保持值域 [0, 1]，与 attention weights 的语义一致
- 更容易调试（值在合理范围内）
- 与单 head 的情况一致（num_groups=1 时，mean=sum）
- 在排序意义上与 sum 等价（mean = sum / num_groups，num_groups 是常数）

### Accumulator 维度

```python
# 对于 GQA
B = batch_size
H_kv = num_key_value_heads  # 8, 不是 32
W = window_size

# Accumulator 维度
accumulator: [B, H_kv, W]  # [B, 8, 32]
token_observation_count: [B, H_kv, W]  # [B, 8, 32]

# 不是 [B, num_heads, W]！
```

## 💻 实现伪代码

### 初始化

```python
def initialize_dual_window_after_prefill(self, prefill_keys, prefill_values):
    """
    Prefill 结束后初始化双窗口
    
    Args:
        prefill_keys: [B, H, N, D] - Prefill 的完整 KV cache（已稀疏化）
        prefill_values: [B, H, N, D]
    
    Returns:
        None (存储到 self.window_state)
    """
    N = prefill_keys.shape[2]
    B, H = prefill_keys.shape[:2]
    W = self.window_size
    
    if N >= W:
        # 情况 A: Prefill >= W
        # 分割：前 N-W 作为已压缩，最后 W 作为 Window A
        compressed_keys = prefill_keys[:, :, :-W, :]
        compressed_values = prefill_values[:, :, :-W, :]
        window_a_keys = prefill_keys[:, :, -W:, :].clone()
        window_a_values = prefill_values[:, :, -W:, :].clone()
        window_a_size = W
    else:
        # 情况 B: Prefill < W
        # 全部作为 Window A（未满）
        compressed_keys = None
        compressed_values = None
        window_a_keys = prefill_keys.clone()
        window_a_values = prefill_values.clone()
        window_a_size = N
    
    # Window B 初始为空
    window_b_keys = None
    window_b_values = None
    window_b_size = 0
    
    # 初始化累积器（注意：H 是 num_key_value_heads，不是 num_heads）
    accumulator = torch.zeros(B, H, W, device=prefill_keys.device, dtype=torch.float32)
    token_observation_count = torch.zeros(B, H, W, device=prefill_keys.device, dtype=torch.float32)
    
    # 存储到实例变量
    self.window_state = {
        'compressed_keys': compressed_keys,
        'compressed_values': compressed_values,
        'window_a_keys': window_a_keys,
        'window_a_values': window_a_values,
        'window_a_size': window_a_size,
        'window_b_keys': window_b_keys,
        'window_b_values': window_b_values,
        'window_b_size': window_b_size,
        'accumulator': accumulator,
        'token_observation_count': token_observation_count
    }
```

### Decode 单步

```python
def decode_step_with_dual_window(self, query_states, new_key, new_value, attention_mask):
    """
    单步 Decode 处理（支持 GQA）
    
    Args:
        query_states: [B, H_q, 1, D] - Query states (num_heads)
        new_key: [B, H_kv, 1, D] - 新 token 的 key (num_key_value_heads)
        new_value: [B, H_kv, 1, D] - 新 token 的 value
        attention_mask: [B, 1, 1, T_total] or None
    
    Returns:
        attn_output: [B, H_q, 1, D]
    """
    B, H_q, _, D = query_states.shape
    _, H_kv, _, _ = new_key.shape
    W = self.window_size
    
    ws = self.window_state  # 简写
    
    # 1. 添加新 token 到 Window A 或 Window B
    if ws['window_a_size'] < W:
        # 情况 B: Window A 未满，添加到 Window A
        if ws['window_a_keys'] is None:
            ws['window_a_keys'] = new_key
            ws['window_a_values'] = new_value
        else:
            ws['window_a_keys'] = torch.cat([ws['window_a_keys'], new_key], dim=2)
            ws['window_a_values'] = torch.cat([ws['window_a_values'], new_value], dim=2)
        ws['window_a_size'] += 1
    else:
        # 情况 A: Window A 已满，添加到 Window B
        if ws['window_b_keys'] is None:
            ws['window_b_keys'] = new_key
            ws['window_b_values'] = new_value
        else:
            ws['window_b_keys'] = torch.cat([ws['window_b_keys'], new_key], dim=2)
            ws['window_b_values'] = torch.cat([ws['window_b_values'], new_value], dim=2)
        ws['window_b_size'] += 1
    
    # 2. 构建完整的 KV cache
    kv_list = []
    if ws['compressed_keys'] is not None:
        kv_list.append((ws['compressed_keys'], ws['compressed_values']))
    if ws['window_a_keys'] is not None:
        kv_list.append((ws['window_a_keys'], ws['window_a_values']))
    if ws['window_b_keys'] is not None:
        kv_list.append((ws['window_b_keys'], ws['window_b_values']))
    
    full_keys = torch.cat([k for k, v in kv_list], dim=2)  # [B, H_kv, T_total, D]
    full_values = torch.cat([v for k, v in kv_list], dim=2)
    
    # 3. 计算 attention (需要 repeat_kv 处理 GQA)
    attn_weights = torch.matmul(
        query_states,  # [B, H_q, 1, D]
        repeat_kv(full_keys, self.num_key_value_groups).transpose(2, 3)  # [B, H_q, T_total, D]
    ) / math.sqrt(self.head_dim)
    # attn_weights: [B, H_q, 1, T_total]
    
    # 应用 mask（Decode 阶段通常是 None 或全 0）
    if attention_mask is not None:
        attn_weights = attn_weights + attention_mask
        attn_weights = torch.max(
            attn_weights, 
            torch.tensor(torch.finfo(attn_weights.dtype).min)
        )
    
    # Softmax
    attn_weights = F.softmax(attn_weights, dim=-1, dtype=torch.float32).to(query_states.dtype)
    
    # 4. 聚合 attention 到 KV head 维度（GQA）
    attn_weights_grouped = attn_weights.view(
        B, H_kv, self.num_key_value_groups, 1, -1
    )  # [B, H_kv, group_size, 1, T_total]
    
    attn_weights_kv = attn_weights_grouped.mean(dim=2)  # [B, H_kv, 1, T_total]
    
    # 5. 累积对 Window A 的 attention
    if ws['window_a_size'] > 0:
        # 计算 Window A 在完整 KV cache 中的位置
        compressed_len = 0 if ws['compressed_keys'] is None else ws['compressed_keys'].shape[2]
        window_a_start = compressed_len
        window_a_end = window_a_start + ws['window_a_size']
        
        # 提取对 Window A 的 attention
        window_a_attention = attn_weights_kv[:, :, 0, window_a_start:window_a_end]  # [B, H_kv, window_a_size]
        
        # 累积
        ws['accumulator'][:, :, :ws['window_a_size']] += window_a_attention
        ws['token_observation_count'][:, :, :ws['window_a_size']] += 1
    
    # 6. 计算 attention output（使用原始的 attn_weights）
    attn_output = torch.matmul(
        attn_weights,  # [B, H_q, 1, T_total]
        repeat_kv(full_values, self.num_key_value_groups)  # [B, H_q, T_total, D]
    )  # [B, H_q, 1, D]
    
    # 7. 检查是否触发压缩
    if ws['window_a_size'] == W and ws['window_b_size'] >= W:
        self._trigger_compression()
    
    return attn_output
```

### 压缩触发

```python
def _trigger_compression(self):
    """
    触发 Window A 的压缩并执行窗口滑动
    """
    ws = self.window_state
    W = self.window_size
    
    # 只有当 Window A 存在且满时才压缩
    if ws['window_a_size'] == W:
        # 1. 归一化 attention（动态计算观察次数）
        normalized_attention = torch.zeros_like(ws['accumulator'])
        
        for i in range(W):
            # 动态计算观察次数
            N_internal = W - 1 - i  # Window A 内部
            N_external = ws['token_observation_count'][:, :, i]  # 实际累积次数
            N_total = N_internal + N_external
            
            # 避免除零
            N_total = torch.clamp(N_total, min=1.0)
            
            normalized_attention[:, :, i] = ws['accumulator'][:, :, i] / N_total
        
        # 2. 使用 Prefill 的 thresholds 分类
        # (self.diff_sparse_thresholds 在 Prefill 阶段已保存)
        if not self.thresholds_computed or self.diff_sparse_thresholds is None:
            raise RuntimeError("Thresholds not computed. Must run prefill first.")
        
        threshold_high, threshold_low = self.diff_sparse_thresholds
        
        # 3. 应用稀疏化
        compressed_keys, compressed_values = self.sparsity_applier.classify_and_apply_sparsity(
            importance_scores=normalized_attention,
            key_states=ws['window_a_keys'],
            value_states=ws['window_a_values'],
            thresholds=(threshold_high, threshold_low)
        )
        
        # 4. 合并到已压缩区域
        if ws['compressed_keys'] is None:
            ws['compressed_keys'] = compressed_keys
            ws['compressed_values'] = compressed_values
        else:
            ws['compressed_keys'] = torch.cat([
                ws['compressed_keys'], compressed_keys
            ], dim=2)
            ws['compressed_values'] = torch.cat([
                ws['compressed_values'], compressed_values
            ], dim=2)
    
    # 5. 窗口滑动: Window B → Window A
    ws['window_a_keys'] = ws['window_b_keys']
    ws['window_a_values'] = ws['window_b_values']
    ws['window_a_size'] = ws['window_b_size']
    
    # 6. 重置 Window B
    ws['window_b_keys'] = None
    ws['window_b_values'] = None
    ws['window_b_size'] = 0
    
    # 7. 重置累积器
    ws['accumulator'].zero_()
    ws['token_observation_count'].zero_()
```

## 📊 生成长度与压缩次数

| 生成长度 | 压缩次数 | 说明 |
|---------|---------|------|
| < 32 | 0 | Window B 未满，不触发压缩 |
| 32 | 1 | Window B 满一次，压缩一次 |
| 33-63 | 1 | Window B 第二次未满 |
| 64 | 2 | Window B 满两次，压缩两次 |
| 96 | 3 | 压缩三次 |
| 128 | 4 | 压缩四次 |

**LongBench 评估**：
- 大部分任务生成长度 < 128 tokens
- 压缩次数通常 0-4 次
- 如果生成 < 32 tokens，双窗口机制不会触发，所有 tokens 保持 dense
