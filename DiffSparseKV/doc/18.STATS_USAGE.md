# DiffSparseKV 稀疏度统计功能使用说明

## 功能概述

稀疏度统计功能可以帮助你验证 DiffSparseKV 算法是否正确工作，通过收集和分析 KV cache 的实际稀疏度。

## 快速开始

### 1. 启用统计功能并运行

```bash
# 启用统计并运行完整评测
export DIFFSPARSE_ENABLE_STATS=1
bash run_diffsparse.sh

# 或者一行命令
DIFFSPARSE_ENABLE_STATS=1 bash run_diffsparse.sh
```

### 2. 只测试单个数据集（可选）

如果想快速测试，可以临时修改数据集列表：

**方法1：修改代码**
编辑 `pred_long_bench_diff_sparse.py`，找到第 260 行左右：

```python
# 将这行
datasets = ["narrativeqa", "qasper", "multifieldqa_en", ...]

# 改为
datasets = ["narrativeqa"]  # 只测试一个数据集
```

然后运行：
```bash
DIFFSPARSE_ENABLE_STATS=1 bash run_diffsparse.sh
```

**方法2：使用现有结果**
如果已经运行过评测，可以直接查看统计：

```bash
# 查看已有的统计文件
cat pred/Meta-Llama-3-8B-Instruct_8192_diff_sparse_kv_0.70/sparsity_statistics.json
```

## 统计内容

### 收集的统计数据

1. **DiffSparseKV 应用后的稀疏度**
   - K cache 稀疏度
   - V cache 稀疏度
   - Token 分类分布 [Level 0, Level 1, Level 2]

2. **最终 KV cache 的稀疏度**
   - 整体 K/V 稀疏度
   - Prefix 稀疏度（前面的 token）
   - Suffix 稀疏度（最后 32 个 token）

运行时会在控制台显示：

```
----------------------------------------------------------------------------------------------------

----------------------------------------------------------------------------------------------------
全局平均               85.0%      84.8%

⚠️  稀疏度异常 (与预期值差异较大)
   预期: 72.5%, 实际: K=85.0%, V=84.8%
   ⚠️  Prefix 被过度稀疏 (K=99.1%)，可能是 dh_prune_key 的 bug
```

## 问题诊断

### 正常情况

- 整体稀疏度接近 72.5%
- Prefix 和 Suffix 稀疏度都接近 72.5%
- Level 分布接近 [0.05, 0.75, 0.20]

### 异常情况1：Prefix 过度稀疏

```
Prefix: K=99.2%, V=99.1%
Suffix: K=72.5%, V=72.6%
```

**原因**：`dh_prune_key` 的逻辑错误，将 `k_sparsity=0.0` 理解为"保留 0% 的元素"

**解决方案**：修复 `models/llama_mustafar_Kt_Mag_Vt_Mag.py` 中的 `dh_prune_key` 方法

### 异常情况2：整体稀疏度过低

```
最终: K=10.5%, V=10.3%
```

**原因**：DiffSparseKV 可能没有被正确调用

**解决方案**：检查模型配置和 forward 方法

### 异常情况3：Level 分布异常

```
Level分布: [0.30, 0.50, 0.20]
```

**原因**：阈值计算可能有问题

**解决方案**：检查 `threshold_manager.py` 的实现

## 代码修改说明

   - 添加了统计收集代码（约 30 行）
   - 添加了 `get_stats()` 和 `reset_stats()` 方法

- 不影响原有功能
- 可以随时禁用

## 高级用法

### 只统计特定数据集

修改 `pred_long_bench_diff_sparse.py` 中的 `datasets` 列表：

```python
# 只测试一个数据集
datasets = ["narrativeqa"]
```

### 分析统计结果

```python
import json
import numpy as np

with open('pred/.../sparsity_statistics.json') as f:
    stats = json.load(f)

# 计算平均值
for dataset, data in stats['per_dataset'].items():
    print(f"{dataset}: K={data['final_k']:.2%}, V={data['final_v']:.2%}")
```

## 常见问题

### Q: 统计会影响性能吗？

A: 影响很小，只增加了少量的稀疏度计算（已经在 DEBUG 模式中存在）。

### Q: 可以只统计部分数据集吗？

A: 可以，修改 `datasets` 列表或使用 `test_stats.sh`。

### Q: 统计文件在哪里？

A: 在输出目录下：`pred/Meta-Llama-3-8B-Instruct_8192_diff_sparse_kv_0.70/sparsity_statistics.json`

### Q: 如何禁用统计？

A: 不设置 `DIFFSPARSE_ENABLE_STATS` 环境变量，或设置为 0。
