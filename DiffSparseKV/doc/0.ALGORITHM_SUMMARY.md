## 核心思想

**DiffSparseKV** 是一种基于 token 重要性的差分稀疏化方法，核心创新点：

### 🎯 主要特点
1. **差分稀疏度**：不同 token 使用不同的稀疏度级别
2. **3级分类系统**：将 token 分为 3 个稀疏度等级
3. **全局阈值**：在 prefill 阶段计算全局阈值，decode 阶段复用
4. **Magnitude Pruning**：基于特征维度的幅度进行剪枝

### 📊 稀疏度配置
```
Level 0: 0%  稀疏度 → 完全保留 (重要 token)
Level 1: 70% 稀疏度 → 保留 30% 特征
Level 2: 100% 稀疏度 → 完全删除 (不重要 token)

目标分布: [5%, 75%, 20%]
预期平均稀疏度: 0.05×0% + 0.75×70% + 0.20×100% = 72.5%
```

## 算法流程

### 🔵 Prefill 阶段（提示词处理）

```
输入: Prompt tokens (长度 T)

Step 1: 前向传播获取 Attention Weights
├─ outputs = model(prompt, output_attentions=True)
├─ attention_weights: [B, H, T_q, T_k]
└─ 已经包含 causal mask 和 softmax

Step 2: 计算 Token 重要性分数
├─ importance_calculator.compute_diffkv_importance(attention_weights)
├─ 输入: [B, H, T_q, T_k] attention weights
├─ 处理:
│   ├─ 创建 causal mask: query i 只能看到 key 0..i
│   ├─ 计算每个 key 被多少个 query 关注
│   ├─ 对每个 key，计算平均 attention 概率
│   └─ 乘以 T_k 进行归一化
└─ 输出: [B, H, T_k] importance scores

Step 3: 计算每层独立的阈值
├─ 对每一层单独计算阈值
├─ threshold_manager.compute_per_layer_thresholds(layer_importance)
├─ 处理:
│   ├─ 输入: 单层的 importance: [B, H, T]
│   ├─ 展平该层的 importance: [B × H × T]
│   ├─ 降序排序
│   ├─ 提取分位点:
│   │   ├─ threshold_high = 前 5% 的边界
│   │   └─ threshold_low = 前 80% 的边界
│   └─ 每层保存各自的阈值
└─ 输出: 每层的 (threshold_high, threshold_low)

Step 4: Token 分类
├─ 对每个 token 的 importance score:
│   ├─ if score >= threshold_high  → Level 0 (0% 稀疏)
│   ├─ elif score >= threshold_low → Level 1 (70% 稀疏)
│   └─ else                        → Level 2 (100% 稀疏)
└─ 输出: [B, H, T] level assignments

Step 5: 应用 Magnitude Pruning
├─ sparsity_applier.classify_and_apply_sparsity(...)
├─ 对每个 token:
│   ├─ Level 0: 保留所有特征
│   ├─ Level 1: 
│   │   ├─ K 独立处理:
│   │   │   ├─ 计算 K 的幅度: k_mag = |K|
│   │   │   ├─ 选择 K 的 top 30% 特征维度
│   │   │   └─ K 的其他维度置零
│   │   └─ V 独立处理:
│   │       ├─ 计算 V 的幅度: v_mag = |V|
│   │       ├─ 选择 V 的 top 30% 特征维度
│   │       └─ V 的其他维度置零
│   └─ Level 2: K 和 V 的所有特征置零
└─ 输出: 稀疏化的 K, V cache

注意: K 和 V 各自基于自己的幅度独立稀疏化

Step 6: 应用 MUSTAFAR 残差处理
├─ 保留最后 residual_length (128) 个 token 不压缩
├─ 对前面的 token 应用 MUSTAFAR 的 dh_prune_key/value
└─ 输出: 最终的 KV cache
```

### 🟢 Decode 阶段（生成阶段）

```
每生成一个新 token:

Step 1: 获取新 token 的 K, V
├─ 计算 query, key, value projections
└─ 应用 rotary embeddings

Step 2: 拼接到历史 KV cache
├─ key_states_full = concat([past_keys, new_key])
└─ value_states_full = concat([past_values, new_value])

Step 3: 计算 Attention
├─ attn_weights = Q @ K^T / sqrt(d)
├─ 应用 causal mask
├─ softmax
└─ attn_output = attn_weights @ V

Step 4: 对新 token 应用稀疏化（可选）
├─ 使用 prefill 阶段的 thresholds
├─ 计算新 token 的 importance
├─ 分类并应用 magnitude pruning
└─ 更新 KV cache 中的新 token

Step 5: 滑动窗口管理
├─ 如果超过 residual_length:
│   ├─ 将超出窗口的 token 进行 MUSTAFAR 压缩
│   └─ 保持最近的 residual_length 个 token 不压缩
└─ 输出: 更新后的 KV cache

Step 6: 生成输出
├─ 投影到词表空间
└─ 采样下一个 token
```

## 关键组件

### 1️⃣ **DiffKVImportanceCalculator** (重要性计算器)

**文件**: `importance_calculator.py`

**核心方法**: `compute_diffkv_importance(attention_weights)`

**算法**:
```python
输入: attention_weights [B, H, T_q, T_k]

# 1. 创建 causal mask
causal_mask = torch.tril(ones(T_q, T_k))  # 下三角矩阵

# 2. 计算每个 key 被多少个 query 关注
valid_queries_per_key = causal_mask.sum(dim=0)  # [T_k]
# key k 可以被 query k, k+1, ..., T_q-1 关注

# 3. 对每个 key，计算总的 attention
masked_attention = attention_weights * causal_mask
importance_sum = masked_attention.sum(dim=2)  # [B, H, T_k]

# 4. 计算平均 attention（归一化）
importance = importance_sum / valid_queries_per_key  # [B, H, T_k]

# 5. 乘以 T_k 进行缩放
importance_scores = importance * T_k

输出: importance_scores [B, H, T_k]
```

**物理意义**:
- 每个 token 的 importance score 表示：在所有能看到它的 query 中，平均有多大概率会关注它
- 乘以 T_k 是为了让分数在合理范围内，便于设置阈值

---

### 2️⃣ **GlobalThresholdManager** (全局阈值管理器)

**文件**: `threshold_manager.py`

**核心方法**: `compute_per_layer_thresholds(importance_scores)`

**算法**:
```python
输入: importance_scores [B, H, T] - 单层的 importance scores

# 1. 展平该层的维度
flattened = importance_scores.flatten()  # [B × H × T]

# 2. 降序排序
sorted_importance = sort(flattened, descending=True)

# 3. 计算分位点索引
n = len(sorted_importance)
percentile_5 = int(0.05 * n)   # 前 5%
percentile_80 = int(0.80 * n)  # 前 80%

# 4. 提取阈值
threshold_high = sorted_importance[percentile_5]   # Level 0 vs Level 1
threshold_low = sorted_importance[percentile_80]   # Level 1 vs Level 2

输出: (threshold_high, threshold_low) - 该层的阈值
```

**设计理念**:
- **Per-layer 阈值**: 每层独立计算阈值，允许层级特定的稀疏模式
- **固定分布**: 通过分位点确保每层的目标分布 [5%, 75%, 20%]
- **Prefill 计算，Decode 复用**: 每层的阈值在 prefill 计算一次，decode 阶段复用

**优势**:
- 不同层的 importance 分布可能差异很大，per-layer 阈值更适应
- 浅层和深层可以有不同的稀疏策略
- 避免全局阈值导致某些层过度稀疏或保留过多

---

### 3️⃣ **SparsityClassifierApplier** (稀疏度分类与应用器)

**文件**: `sparsity_applier.py`

**核心方法**: `classify_and_apply_sparsity(...)`

**算法**:
```python
输入: 
  - importance_scores [B, H, T]
  - key_states [B, H, T, D]
  - value_states [B, H, T, D]
  - thresholds (threshold_high, threshold_low)

# Step 1: 分类 token
levels = torch.full_like(importance_scores, 2)  # 初始化为 Level 2
levels[importance_scores >= threshold_low] = 1   # Level 1
levels[importance_scores >= threshold_high] = 0  # Level 0

# Step 2: 对每个 level 应用稀疏化
for level in [0, 1, 2]:
    sparsity_rate = sparsity_levels[level]  # [0.0, 0.7, 1.0]
    level_mask = (levels == level)  # 找到该 level 的 token
    
    if sparsity_rate == 0.0:
        # Level 0: 不做任何处理
        continue
    
    elif sparsity_rate >= 1.0:
        # Level 2: 完全删除
        key_states[level_mask] = 0.0
        value_states[level_mask] = 0.0
    
    else:
        # Level 1: Magnitude-based Top-K (K 和 V 独立处理)
        for each token in level_mask:
            # 1. K 独立处理
            k_mag = abs(key_states[token])  # [D]
            keep_ratio = 1 - sparsity_rate  # 0.3 for 70% sparsity
            num_keep = int(keep_ratio * D)
            k_top_indices = topk(k_mag, num_keep)
            
            k_keep_mask = zeros(D, dtype=bool)
            k_keep_mask[k_top_indices] = True
            key_states[token, ~k_keep_mask] = 0.0
            
            # 2. V 独立处理
            v_mag = abs(value_states[token])  # [D]
            v_top_indices = topk(v_mag, num_keep)
            
            v_keep_mask = zeros(D, dtype=bool)
            v_keep_mask[v_top_indices] = True
            value_states[token, ~v_keep_mask] = 0.0

输出: (pruned_keys, pruned_values)
```

**关键设计**:
- **K 和 V 独立稀疏**: K 和 V 各自基于自己的幅度独立选择保留的维度
- **Magnitude-based**: 保留幅度最大的特征，假设它们最重要
- **In-place 操作**: 直接修改张量，节省内存

### 4️⃣ **LlamaDiffSparseKVAttention** (Llama 注意力层集成)

**文件**: `llama_diff_sparse_kv.py`

**核心流程**:
```python
class LlamaDiffSparseKVAttention:
    def forward(...):
        if is_prefill_stage:
            return self._prefill_with_diff_sparse(...)
        else:
            return self._decode_with_diff_sparse(...)
    
    def _prefill_with_diff_sparse(...):
        # 1. 计算 attention weights
        attn_weights = Q @ K^T / sqrt(d)
        attn_weights = softmax(attn_weights + causal_mask)
        
        # 2. 计算 importance
        importance = importance_calculator.compute_diffkv_importance(attn_weights)
        
        # 3. 计算全局阈值
        thresholds = threshold_manager.compute_global_thresholds(importance)
        self.diff_sparse_thresholds = thresholds  # 保存供 decode 使用
        
        # 4. 应用稀疏化
        sparse_K, sparse_V = sparsity_applier.classify_and_apply_sparsity(
            importance, K, V, thresholds
        )
        
        # 5. 应用 MUSTAFAR 残差处理
        K_full, V_full = apply_mustafar_residual(sparse_K, sparse_V)
        
        # 6. 计算输出
        attn_output = flash_attention(Q, K_full, V_full)
        return attn_output, past_kv_cache
    
    def _decode_with_diff_sparse(...):
        # 1. 拼接新 token 到历史 KV
        K_full = concat([past_K, new_K])
        V_full = concat([past_V, new_V])
        
        # 2. 计算 attention
        attn_output = compute_attention(Q, K_full, V_full)
        
        # 3. 对新 token 应用稀疏化（使用 prefill 的阈值）
        if self.diff_sparse_thresholds is not None:
            new_importance = compute_importance(new_token_attention)
            sparse_new_K, sparse_new_V = apply_sparsity(
                new_importance, new_K, new_V, self.diff_sparse_thresholds
            )
            K_full[:, :, -1:, :] = sparse_new_K
            V_full[:, :, -1:, :] = sparse_new_V
        
        # 4. 滑动窗口管理
        if len(K_full) > residual_length:
            K_full = apply_window_compression(K_full)
            V_full = apply_window_compression(V_full)
        
        return attn_output, (K_full, V_full)
```

## 实现细节

### 🔧 技术细节

#### 1. Causal Attention 处理
```python
# 为什么需要 causal mask？
# 在自回归生成中，query i 只能看到 key 0..i
# 所以计算 importance 时要考虑这个约束

causal_mask = torch.tril(torch.ones(T_q, T_k))
# [[1, 0, 0, 0],
#  [1, 1, 0, 0],
#  [1, 1, 1, 0],
#  [1, 1, 1, 1]]

# key 0 被所有 query 看到 (4 个)
# key 1 被 query 1,2,3 看到 (3 个)
# key 2 被 query 2,3 看到 (2 个)
# key 3 被 query 3 看到 (1 个)
```

#### 2. Magnitude Pruning 的一致性
```python
# 为什么要用平均幅度？
# 确保 K 和 V 的稀疏模式一致

k_mag = torch.abs(key_vector)    # [D]
v_mag = torch.abs(value_vector)  # [D]
avg_mag = (k_mag + v_mag) / 2    # [D]

# 基于 avg_mag 选择 top-k
# 然后同时对 K 和 V 的相同维度置零
# 这样 K 和 V 的非零位置完全一致
```

#### 3. 数值稳定性
```python
# 1. Epsilon 防止除零
valid_queries_per_key = torch.clamp(valid_queries_per_key, min=1)

# 2. 检查 NaN 和 Inf
if torch.any(torch.isnan(attention_weights)):
    raise ValueError("NaN detected")

# 3. Clamp importance scores
importance_scores = torch.clamp(importance_scores, min=0.0, max=float(T_k))
```

#### 4. 与 MUSTAFAR 的集成
```python
# DiffSparseKV 继承自 MUSTAFAR
# 复用 MUSTAFAR 的:
# - Flash Attention 实现
# - Residual length 窗口管理
# - dh_prune_key/value 压缩方法

# 在 DiffSparseKV 之后应用 MUSTAFAR:
sparse_K, sparse_V = apply_diff_sparse_kv(K, V)
final_K, final_V = apply_mustafar_residual(sparse_K, sparse_V)
```

## 与其他方法的对比

### 📊 对比表

| 方法 | 稀疏度类型 | 阈值计算 | Token 级别 | 特征级别 |
|------|-----------|---------|-----------|---------|
| **Uniform** | 统一稀疏 | 固定比例 | ❌ 所有 token 相同 | ✅ Magnitude |
| **H2O** | 二值稀疏 | 动态计算 | ✅ 保留/删除 | ❌ 全保留或全删除 |
| **SnapKV** | 观察窗口 | 基于窗口 | ✅ 窗口内外 | ❌ 全保留或全删除 |
| **PyramidKV** | 层级稀疏 | Per-layer | ✅ 不同层不同 | ❌ 全保留或全删除 |
| **DiffSparseKV** | 差分稀疏 | 全局阈值 | ✅ 3 级分类 | ✅ Magnitude |

### 🎯 DiffSparseKV 的优势

1. **细粒度控制**: 3 级稀疏度 vs 二值（保留/删除）
2. **特征级剪枝**: 70% 稀疏保留 30% 重要特征，而不是完全删除
3. **全局一致性**: 所有层使用相同阈值，避免层间不一致
4. **固定阈值**: Prefill 计算一次，decode 复用，效率高

### ⚠️ 潜在问题

1. **阈值泛化性**: Prefill 的阈值可能不适用于 decode
2. **100% 稀疏太激进**: 20% 的 token 完全删除可能损失信息
3. **计算开销**: 需要计算 attention weights 来获取 importance
4. **固定分布**: [5%, 75%, 20%] 可能不是所有任务的最优分布

## 📝 总结

**DiffSparseKV 的核心创新**:
1. 基于 token 重要性的差分稀疏化
2. 3 级稀疏度系统（0%, 70%, 100%）
3. **Per-layer 阈值管理**（每层独立计算阈值，prefill 计算，decode 复用）
4. **K 和 V 独立稀疏化**（各自基于自己的 magnitude 进行特征级剪枝）

**关键设计决策**:
- ✅ **Per-layer 阈值**: 每层独立计算，适应不同层的 importance 分布
- ✅ **K/V 独立**: K 和 V 各自选择保留的特征维度，更灵活
- ✅ **Magnitude-based**: 保留幅度最大的特征，简单有效

**适用场景**:
- 长文本生成任务
- 需要精细控制内存和精度平衡
- 不同 token 重要性差异大的任务
- 不同层的 attention 模式差异大的模型

**与 MUSTAFAR 的区别**:
- MUSTAFAR: 统一稀疏度，所有 token 相同处理
- DiffSparseKV: 差分稀疏度，基于 importance 的 3 级分类
- 两者都使用 magnitude-based pruning 和 per-layer 处理

**未来改进方向**:
- 动态调整稀疏度级别（根据任务自适应）
- Per-head 的阈值（不同 attention head 可能需要不同策略）
- 更智能的 importance 计算方法（考虑 token 的语义信息）
- 在线调整阈值（decode 阶段根据生成质量动态调整）
