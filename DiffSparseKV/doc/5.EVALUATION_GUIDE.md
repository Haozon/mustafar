## 稀疏度统计解读

### 统计文件结构

```json
{
  "per_dataset": {
    "narrativeqa": {
      "final_k": 0.725,
      "final_v": 0.720,
      "prefix_k": 0.750,
      "prefix_v": 0.745,
      "suffix_k": 0.000,
      "suffix_v": 0.000,
      "sample_count": 200
    },
    ...
  },
  "config": {
    "target_distribution": [0.05, 0.75, 0.20],
    "sparsity_levels": [0.0, 0.7, 1.0],
    "expected_avg_sparsity": 0.725,
    "residual_length": 32
  }
}
```

### 字段说明

- **final_k/v**: 最终的 K/V 缓存稀疏度
- **prefix_k/v**: Prefill 阶段的稀疏度
- **suffix_k/v**: 最后 W tokens 的稀疏度（应该为 0，保持 dense）
- **sample_count**: 处理的样本数量

### 正常范围

- **final_k/v**: 应接近 `expected_avg_sparsity` (±5%)
- **prefix_k/v**: 通常略高于 final
- **suffix_k/v**: 应为 0（最后 W tokens 保持 dense）

## 故障排除

### 问题 1: 导入错误

```
ModuleNotFoundError: No module named 'llama_diff_sparse_kv'
```

**解决方案**:
```bash
cd DiffSparseKV
export PYTHONPATH="$(dirname $PWD):$PYTHONPATH"
python test_setup.py
```

### 问题 2: CUDA 内存不足

```
RuntimeError: CUDA out of memory
```

**解决方案**:
1. 减小 batch size（在代码中修改）
2. 使用更小的窗口大小：`--window-size 16`
3. 使用 CPU（不推荐，非常慢）

### 问题 3: 稀疏度异常

如果稀疏度统计显示异常（如 prefix 被过度稀疏）：

**检查**:
1. 查看 DEBUG 输出：`--debug`
2. 检查 threshold 计算是否正确
3. 验证 importance scores 的值域

### 问题 4: 评估分数异常低

如果 DiffSparseKV 分数远低于 Baseline：

**检查**:
1. 窗口大小是否合适（推荐 32-64）
2. 稀疏度是否过高（推荐 70%-80%）
3. 查看 DEBUG 输出确认压缩是否正常触发

### 问题 5: 生成速度慢

**优化建议**:
1. 禁用统计收集：`--no-stats`
2. 禁用调试输出（默认已禁用）
3. 使用更大的 GPU

## 高级用法

### 1. 并行运行多个配置

```bash
# 终端 1
bash run_diffsparse.sh --window-size 32 &

# 终端 2
bash run_diffsparse.sh --window-size 64 &
```

### 2. 只运行特定数据集

修改 `pred_long_bench_diff_sparse.py` 中的 `datasets` 列表：

```python
# 只运行 QA 任务
datasets = ["narrativeqa", "qasper", "multifieldqa_en"]
```

### 3. 批量实验

创建脚本 `run_experiments.sh`:

```bash
#!/bin/bash

for window_size in 16 32 64 128; do
    echo "Running with window_size=$window_size"
    bash run_diffsparse.sh --window-size $window_size
done
```
