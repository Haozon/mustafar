## 🎯 算法理解（基于论文 Section 4.2）

### Decode 阶段的完整流程

```
初始状态（Prefill 结束后）:
┌─────────────────────────────────────────────────────────────┐
│ Prefill Cache (已压缩) │ 最后 W=32 tokens → Window B      │
└─────────────────────────────────────────────────────────────┘
                            Window A: 空
                            Window B: 32 tokens (Prefill 末尾)
                            Accumulator: 初始化为 0

Decode 循环（每生成一个 token）:
Step 1-32:
  - 生成新 token → 追加到 Window B
  - 计算 attention: α_t = softmax(q_t * K_A^T / sqrt(d))
  - 累积: Acc[i] += α_t[i]  (对 Window A 的 attention)
  - Window B 逐渐填满: 1, 2, 3, ..., 32 tokens

Step 32 (Window B 满):
  - 触发压缩事件
  - 归一化: ᾱ^(i) = Acc[i] / (2W - 1 - i)
  - 分类: 使用 Prefill 的 (τ_high, τ_low)
  - 稀疏化 Window A: Level 0/1/2
  - 窗口滑动:
    * Window A (压缩后) → 已压缩区域
    * Window B → 新的 Window A
    * Window B 重置为空
    * Accumulator 重置为 0

Step 33-64:
  - 重复上述过程
  - 新 tokens → Window B
  - 累积对新 Window A 的 attention
  ...
```

### 关键公式（论文中的）

**1. 观察次数（论文 Eq. 4.2）**:
```
N_total^(i) = N_internal^(i) + N_external^(i)
            = (W - 1 - i) + W
            = 2W - 1 - i

其中:
- N_internal^(i) = W - 1 - i  (Window A 内部因果 attention)
- N_external^(i) = W          (Window B 的所有 tokens)
```

**2. 归一化重要性分数（论文 Eq. 4.5）**:
```
ᾱ^(i) = Acc[:, :, i] / (2W - 1 - i)
```

**3. Token 分类（论文 Eq. 4.6）**:
```
Level(i) = {
  0,  if ᾱ^(i) >= τ_high     (保留全部)
  1,  if τ_low <= ᾱ^(i) < τ_high  (保留 30%)
  2,  if ᾱ^(i) < τ_low       (完全删除)
}
```

## 🔧 实现细节

### 1. 初始化（Prefill 结束后）

```python
def initialize_dual_window_after_prefill(self, key_states_full, value_states_full):
    """
    Prefill 结束后初始化双窗口
    
    Args:
        key_states_full: [B, H, seq_len, D] - Prefill 后的完整 KV cache
        value_states_full: [B, H, seq_len, D]
    """
    seq_len = key_states_full.shape[2]
    
    # 保留最后 W 个 tokens 作为初始 Window B
    if seq_len >= self.window_size:
        window_b_keys = key_states_full[:, :, -self.window_size:, :].clone()
        window_b_values = value_states_full[:, :, -self.window_size:, :].clone()
        
        # 前面的作为已压缩区域
        compressed_keys = key_states_full[:, :, :-self.window_size, :]
        compressed_values = value_states_full[:, :, :-self.window_size, :]
    else:
        # 序列太短，全部作为 Window B
        window_b_keys = key_states_full.clone()
        window_b_values = value_states_full.clone()
        compressed_keys = None
        compressed_values = None
    
    # Window A 初始为空
    window_a_keys = None
    window_a_values = None
    
    # 初始化累积器
    B, H = key_states_full.shape[:2]
    accumulator = torch.zeros(B, H, self.window_size, 
                             device=key_states_full.device,
                             dtype=torch.float32)
    
    return {
        'compressed_keys': compressed_keys,
        'compressed_values': compressed_values,
        'window_a_keys': window_a_keys,
        'window_a_values': window_a_values,
        'window_b_keys': window_b_keys,
        'window_b_values': window_b_values,
        'accumulator': accumulator,
        'accumulation_count': 0,
        'window_a_size': 0,
        'window_b_size': window_b_keys.shape[2]
    }
```

### 2. Decode 单步处理

```python
def decode_step_with_dual_window(self, 
                                 query_states,      # [B, H, 1, D]
                                 key_state,         # [B, H, 1, D] - 新 token
                                 value_state,       # [B, H, 1, D] - 新 token
                                 window_state,      # 窗口状态字典
                                 attention_mask):
    """
    单步 Decode 处理
    
    Returns:
        attn_output: [B, H, 1, D]
        updated_window_state: 更新后的窗口状态
    """
    B, H, _, D = query_states.shape
    
    # 1. 添加新 token 到 Window B
    if window_state['window_b_keys'] is None:
        window_state['window_b_keys'] = key_state
        window_state['window_b_values'] = value_state
    else:
        window_state['window_b_keys'] = torch.cat([
            window_state['window_b_keys'], key_state
        ], dim=2)
        window_state['window_b_values'] = torch.cat([
            window_state['window_b_values'], value_state
        ], dim=2)
    
    window_state['window_b_size'] += 1
    
    # 2. 构建完整的 KV cache
    kv_list = []
    if window_state['compressed_keys'] is not None:
        kv_list.append((window_state['compressed_keys'], 
                       window_state['compressed_values']))
    if window_state['window_a_keys'] is not None:
        kv_list.append((window_state['window_a_keys'], 
                       window_state['window_a_values']))
    kv_list.append((window_state['window_b_keys'], 
                   window_state['window_b_values']))
    
    full_keys = torch.cat([k for k, v in kv_list], dim=2)
    full_values = torch.cat([v for k, v in kv_list], dim=2)
    
    # 3. 计算 attention
    attn_weights = torch.matmul(
        query_states, 
        repeat_kv(full_keys, self.num_key_value_groups).transpose(2, 3)
    ) / math.sqrt(self.head_dim)
    
    # 应用 mask
    if attention_mask is not None:
        attn_weights = attn_weights + attention_mask
        attn_weights = torch.max(
            attn_weights, 
            torch.tensor(torch.finfo(attn_weights.dtype).min)
        )
    
    # Softmax
    attn_weights = F.softmax(attn_weights, dim=-1, dtype=torch.float32).to(query_states.dtype)
    
    # 4. 累积对 Window A 的 attention（如果 Window A 存在）
    if window_state['window_a_size'] > 0:
        # 计算 Window A 在完整 KV cache 中的起始位置
        compressed_len = 0 if window_state['compressed_keys'] is None else window_state['compressed_keys'].shape[2]
        window_a_start = compressed_len
        window_a_end = window_a_start + window_state['window_a_size']
        
        # 提取对 Window A 的 attention
        window_a_attention = attn_weights[:, :, 0, window_a_start:window_a_end]  # [B, H, W]
        
        # 累积
        window_state['accumulator'][:, :, :window_state['window_a_size']] += window_a_attention
        window_state['accumulation_count'] += 1
    
    # 5. 计算 attention output
    attn_output = torch.matmul(
        attn_weights, 
        repeat_kv(full_values, self.num_key_value_groups)
    )
    
    # 6. 检查是否需要触发压缩
    if window_state['window_b_size'] >= self.window_size:
        window_state = self._trigger_compression(window_state)
    
    return attn_output, window_state
```

### 3. 压缩触发

```python
def _trigger_compression(self, window_state):
    """
    触发 Window A 的压缩并执行窗口滑动
    """
    # 只有当 Window A 存在时才压缩
    if window_state['window_a_size'] > 0:
        # 归一化 attention
        W = self.window_size
        normalized_attention = torch.zeros_like(window_state['accumulator'])
        
        for i in range(window_state['window_a_size']):
            # 论文公式: ᾱ^(i) = Acc[i] / (2W - 1 - i)
            observation_count = 2 * W - 1 - i
            normalized_attention[:, :, i] = window_state['accumulator'][:, :, i] / observation_count
        
        # 使用 Prefill 的 thresholds 分类
        # (self.diff_sparse_thresholds 在 Prefill 阶段已保存)
        threshold_high, threshold_low = self.diff_sparse_thresholds
        
        # 应用稀疏化
        compressed_keys, compressed_values = self.sparsity_applier.classify_and_apply_sparsity(
            importance_scores=normalized_attention[:, :, :window_state['window_a_size']],
            key_states=window_state['window_a_keys'],
            value_states=window_state['window_a_values'],
            thresholds=(threshold_high, threshold_low)
        )
        
        # 合并到已压缩区域
        if window_state['compressed_keys'] is None:
            window_state['compressed_keys'] = compressed_keys
            window_state['compressed_values'] = compressed_values
        else:
            window_state['compressed_keys'] = torch.cat([
                window_state['compressed_keys'], compressed_keys
            ], dim=2)
            window_state['compressed_values'] = torch.cat([
                window_state['compressed_values'], compressed_values
            ], dim=2)
    
    # 窗口滑动: Window B → Window A
    window_state['window_a_keys'] = window_state['window_b_keys']
    window_state['window_a_values'] = window_state['window_b_values']
    window_state['window_a_size'] = window_state['window_b_size']
    
    # 重置 Window B
    window_state['window_b_keys'] = None
    window_state['window_b_values'] = None
    window_state['window_b_size'] = 0
    
    # 重置累积器
    window_state['accumulator'].zero_()
    window_state['accumulation_count'] = 0
    
    return window_state
```

**关键区别**:
- MUSTAFAR: 每次 prune **1 个** token
- DiffSparseKV: 每 32 tokens 批量 prune **32 个** tokens

## ❓ 需要确认的问题

### 1. Window 大小参数
- ✅ 单个窗口: 32 tokens（默认，用户可配置）
- ✅ 总窗口: 64 tokens (Window A + Window B)
- ✅ 与 MUSTAFAR 的 residual_length=32 一致

### 2. 初始化方式
根据论文，Prefill 结束后：
- Window B: Prefill 末尾的 W=32 tokens
- Window A: 空（等待 Window B 滑动过来）

### 3. Flash Attention
根据论文，Decode 阶段需要 attention weights 来累积，因此：
- **必须禁用 Flash Attention**（在 decode 阶段）
- 手动计算 attention 以获取 weights

### 4. Threshold 复用
根据论文 Section 4.2.2：
- Decode 阶段**复用** Prefill 阶段的 (τ_high, τ_low)
- **不重新计算** thresholds

### 5. past_key_value 格式
需要保存窗口状态：
```python
past_key_value = (
    full_keys,           # 完整的 KV cache（用于计算）
    full_values,
    kv_seq_len,
    window_state_dict    # 窗口状态
)
```
