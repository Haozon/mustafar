# DiffSparseKV 观察窗口优化方案

## 核心思想

观察窗口优化解决了 KV 缓存压缩中速度与准确性的根本权衡问题。

### 问题背景

**当前问题**：
- 计算完整注意力矩阵 [seq_len, seq_len] 非常慢（prefill 阶段需要 18-20 秒）
- 无法使用 Flash Attention（因为需要 attention weights 来计算重要性）
- 导致推理速度慢，无法实际应用

### 解决方案

使用**观察窗口**方法，灵感来自 SnapKV，但适配 DiffSparseKV 的架构：

1. **观察窗口（Observation Window）**：使用序列末尾的 N 个 token（默认 128）作为查询来计算注意力权重
2. **Window A**：序列末尾的 M 个 token（默认 32）绝对不压缩
3. **压缩范围**：Window A 之前的所有 token，包括观察窗口的前半部分

## 窗口关系图解

```
完整序列 (假设 seq_len = 2048):
┌─────────────────────────────────────────────────────────────┐
│ [0 -------- 2015]  [2016 -------- 2047]                      │
│       ↑                    ↑                                  │
│   压缩范围              Window A                              │
│   (2016 tokens)        (32 tokens)                           │
│                        绝对不压缩                             │
│                                                               │
│   [1920 -------- 2015]  [2016 -------- 2047]                 │
│          ↑                      ↑                             │
│    观察窗口前部            Window A                           │
│    (96 tokens)           (32 tokens)                         │
│    会被压缩               不压缩                              │
│                                                               │
│   └──────────── 观察窗口 (128 tokens) ─────────────┘         │
│                  用于计算重要性                               │
└─────────────────────────────────────────────────────────────┘
```

### 关键点

1. **观察窗口** = `[seq_len - obs_window_size : seq_len]` = `[1920:2048]`
   - 用于**计算**所有 token 的重要性分数
   - 大小：128 tokens（默认）

2. **Window A** = `[seq_len - window_size : seq_len]` = `[2016:2048]`
   - **保护**最近的 token 不被压缩
   - 大小：32 tokens（默认）
   - 是观察窗口的最后一部分

3. **压缩范围** = `[0 : seq_len - window_size]` = `[0:2016]`
   - 包括观察窗口的前 96 个 token `[1920:2016]`
   - 这些 token 虽然参与了重要性计算，但仍然会被压缩

## 算法流程

### Prefill 阶段

```python
def prefill_with_observation_window(query, key, value, obs_window_size=128, window_size=32):
    """
    使用观察窗口优化的 Prefill 阶段
    
    Args:
        query: [B, H_q, seq_len, D] - 查询状态
        key: [B, H_kv, seq_len, D] - 键状态
        value: [B, H_kv, seq_len, D] - 值状态
        obs_window_size: 观察窗口大小（默认 128）
        window_size: Window A 大小（默认 32）
    """
    seq_len = query.shape[2]
    
    # 步骤 1: 使用 Flash Attention 计算主输出（快速，20x 加速）
    attn_output = flash_attention(query, key, value, causal=True)
    
    # 步骤 2: 单独计算观察窗口的注意力（用于重要性计算）
    obs_queries = query[:, :, -obs_window_size:, :]  # 最后 128 个 token
    
    # 使用 GQA 优化的方法计算注意力
    obs_attn_weights = compute_obs_window_attention_gqa(
        obs_queries, key, num_key_value_groups
    )  # [B, H_kv, obs_window_size, seq_len]
    
    # 步骤 3: 计算重要性分数
    importance_scores = compute_diffkv_importance(obs_attn_weights)
    # 返回: [B, H_kv, seq_len] - 所有 token 的重要性
    
    # 步骤 4: 计算阈值
    threshold_high, threshold_low = compute_thresholds(importance_scores)
    
    # 步骤 5: 应用稀疏化
    # 关键：只压缩 Window A 之前的 token
    compress_end = seq_len - window_size  # 2016
    
    # 压缩 [0:2016] 的 token（包括观察窗口前部）
    compressed_keys, compressed_values = apply_sparsity(
        key[:, :, :compress_end, :],
        value[:, :, :compress_end, :],
        importance_scores[:, :, :compress_end],
        thresholds=(threshold_high, threshold_low)
    )
    
    # Window A [2016:2048] 保持密集
    window_a_keys = key[:, :, -window_size:, :]
    window_a_values = value[:, :, -window_size:, :]
    
    # 拼接最终的 KV cache
    final_keys = concat([compressed_keys, window_a_keys], dim=2)
    final_values = concat([compressed_values, window_a_values], dim=2)
    
    return attn_output, final_keys, final_values
```

## GQA 优化策略

对于 Grouped Query Attention (GQA)，使用**重塑 queries 后计算**的方法：

```python
def compute_obs_window_attention_gqa(obs_queries, keys, num_key_value_groups):
    """
    GQA 优化的观察窗口注意力计算
    
    Args:
        obs_queries: [B, H_q, obs_window, D] - 观察窗口的查询
        keys: [B, H_kv, seq_len, D] - 所有键
        num_key_value_groups: 每个 KV head 对应的 query head 数量
    
    Returns:
        attn_weights_kv: [B, H_kv, obs_window, seq_len]
    """
    B, H_q, obs_window, D = obs_queries.shape
    H_kv = keys.shape[1]
    seq_len = keys.shape[2]
    
    # 步骤 1: 重塑 queries 为 KV head 分组
    obs_queries_grouped = obs_queries.view(
        B, H_kv, num_key_value_groups, obs_window, D
    )  # [B, H_kv, group_size, obs_window, D]
    
    # 步骤 2: 计算注意力（利用广播机制）
    attn_weights = torch.matmul(
        obs_queries_grouped,
        keys.unsqueeze(2).transpose(-2, -1)
    ) / math.sqrt(D)
    # 结果: [B, H_kv, group_size, obs_window, seq_len]
    
    # 步骤 3: 应用因果掩码
    causal_mask = create_causal_mask(obs_window, seq_len)
    attn_weights = attn_weights.masked_fill(causal_mask == 0, float('-inf'))
    
    # 步骤 4: Softmax
    attn_weights = F.softmax(attn_weights, dim=-1)
    
    # 步骤 5: 在 group 维度上平均
    attn_weights_kv = attn_weights.mean(dim=2)
    # 结果: [B, H_kv, obs_window, seq_len]
    
    return attn_weights_kv
```

### 为什么使用这个方法？

**优点**：
1. ✅ **内存高效**：不需要扩展 keys，节省显存
2. ✅ **计算高效**：利用广播机制，减少冗余计算
3. ✅ **数值稳定**：避免重复的浮点运算
4. ✅ **可扩展性**：对于更大的模型优势更明显

**性能对比**（假设 H_q=32, H_kv=8, seq_len=2048）：
- 传统方法：需要扩展 keys，额外内存 ~32 MB
- 优化方法：无需扩展，节省 ~32 MB

## 参数配置

```python
# 配置示例
config.obs_window_size = 128      # 观察窗口大小（新参数）
config.window_size = 32           # Window A/B 大小（现有参数）

# 约束条件
assert config.obs_window_size >= config.window_size, \
    "观察窗口必须 >= Window A 大小"
```

### 参数选择建议

**obs_window_size**：
- 默认：128（平衡速度和准确性）
- 追求速度：64
- 追求准确性：256
- ⚠️ 不建议 < 64（统计意义不足）

**window_size**：
- 默认：32（与现有实现一致）
- 可根据任务调整：16, 32, 64

## 性能预期

### 速度提升

- **当前实现**：18-20 秒（prefill 阶段）
- **观察窗口优化**：~1 秒（prefill 阶段）
- **加速比**：~20x

### 准确性保持

- 观察窗口（128 tokens）足以捕获重要性分布
- 与完整注意力计算的重要性分数高度相关
- LongBench 评估指标预期保持相近

## 实现要点

### 1. Flash Attention 依赖

```python
try:
    from flash_attn import flash_attn_func
except ImportError:
    raise ImportError(
        "Flash Attention is required for observation window mode. "
        "Install with: pip install flash-attn"
    )
```

### 2. 因果掩码处理

观察窗口在序列末尾，需要特殊处理因果掩码：

```python
# 创建因果掩码
causal_mask = torch.tril(torch.ones(obs_window, seq_len))

# 调整掩码位置（观察窗口在末尾）
offset = seq_len - obs_window
causal_mask[:, :offset] = 0  # 观察窗口不能看到之前的 token
```

### 3. 边缘情况

```python
# 情况 1: 序列太短
obs_window_size = min(self.obs_window_size, seq_len)

# 情况 2: 不需要压缩
compress_end = seq_len - self.window_size
if compress_end <= 0:
    # 序列太短，不压缩
    return key_states, value_states
```
