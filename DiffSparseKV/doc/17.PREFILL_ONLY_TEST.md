# Prefill-Only 稀疏化测试

## 目的

隔离问题：判断性能下降是由 **prefill 阶段的稀疏化** 还是 **decode 阶段的 dual-window 机制** 导致的。

## 修改内容

### 修改文件
`DiffSparseKV/llama_diff_sparse_kv.py`

### 修改说明

**原来的实现**：
- Prefill: 应用稀疏化 + 初始化 dual-window
- Decode: 使用 dual-window 机制，动态累积 attention 并压缩

**修改后的实现**：
- Prefill: 应用稀疏化（保持不变）
- Decode: **简化为标准 attention**，不使用 dual-window，不进行额外稀疏化

### 代码变化

```python
def _decode_with_diff_sparse(self, ...):
    """
    Decode stage - SIMPLIFIED VERSION
    
    只使用 prefill 阶段生成的稀疏化 KV cache，
    但不在 decode 阶段应用额外的稀疏化。
    """
    # 1. 获取 past KV cache
    past_keys, past_values, past_length = past_key_value
    
    # 2. 拼接新 token
    key_states = torch.cat([past_keys, key_state], dim=2)
    value_states = torch.cat([past_values, value_state], dim=2)
    
    # 3. 标准 attention 计算
    attn_weights = Q @ K^T / sqrt(d)
    attn_weights = softmax(attn_weights)
    attn_output = attn_weights @ V
    
    # 4. 返回更新后的 KV cache
    return attn_output, None, (key_states, value_states, kv_seq_len)
```

原来的 dual-window 实现被重命名为 `_decode_with_diff_sparse_ORIGINAL`，暂时不使用。

## 预期结果

### 情况 1：效果明显改善
**结论**：问题在 **decode 阶段的 dual-window 机制**

**可能原因**：
- Dual-window 的累积逻辑有问题
- 压缩触发条件不合理
- Window sliding 导致信息丢失
- Accumulator 的归一化有问题

**下一步**：
- 检查 `decode_step_with_dual_window` 的实现
- 检查 `_trigger_compression` 的逻辑
- 验证 accumulator 和 observation count 的计算

### 情况 2：效果仍然很差
**结论**：问题在 **prefill 阶段的稀疏化**

**可能原因**：
- 阈值计算有问题（虽然 token 选择看起来合理）
- Magnitude pruning 的实现有问题
- 稀疏化后的 KV cache 质量太差
- 特征级剪枝（70% 稀疏）损失太大

**下一步**：
- 测试不使用 Level 1（70% 稀疏），只用 Level 0 和 Level 2
- 检查 magnitude pruning 的实现
- 验证稀疏化后的 KV cache 是否正确

### 情况 3：效果略有改善但仍不理想
**结论**：**两个阶段都有问题**

**下一步**：
- 先修复 decode 阶段
- 再优化 prefill 阶段
