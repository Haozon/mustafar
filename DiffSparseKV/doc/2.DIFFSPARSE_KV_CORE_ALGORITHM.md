# DiffSparseKV 核心算法技术文档

## 1. 算法概述

DiffSparseKV 是一种基于差异化稀疏性的 KV Cache 压缩算法，通过将 tokens 分为 3 个稀疏度等级来实现高效的内存压缩和推理加速。

### 1.1 核心思想

- **差异化稀疏性**：不同重要性的 tokens 应用不同的稀疏度
- **3 级分类**：将 tokens 分为高重要性（0% 稀疏）、中等重要性（70% 稀疏）、低重要性（100% 稀疏）
- **独立稀疏化**：K 和 V 基于各自的 magnitude 独立进行稀疏化
- **Per-layer 阈值**：每层独立计算阈值，适应层级特性

### 1.2 目标分布

默认配置：
- **Level 0**：5% tokens，0% 稀疏度（完全保留）
- **Level 1**：75% tokens，70% 稀疏度（保留 30% 最大元素）
- **Level 2**：20% tokens，100% 稀疏度（完全移除）

## 2. 算法流程

### 2.1 整体流程图

```
输入: Query, Key, Value, Attention Mask
    ↓
[1] 计算 Attention Weights
    ↓
[2] 计算 DiffKV Importance Scores
    ↓
[3] 计算 Per-Layer Thresholds
    ↓
[4] Token 分类（3 个等级）
    ↓
[5] 应用 Magnitude-based Pruning
    ↓
[6] 应用 MUSTAFAR Residual Processing
    ↓
输出: Sparse Key, Sparse Value
```

### 2.2 Prefill 阶段流程

```python
# 伪代码
def prefill_stage(query, key, value, attention_mask):
    # Step 1: 计算 attention weights
    attn_weights = compute_attention(query, key)  # [B, H, T_q, T_k]
    
    # Step 2: 计算重要性分数
    importance = compute_diffkv_importance(attn_weights)  # [B, H, T_k]
    
    # Step 3: 计算 per-layer 阈值
    threshold_high, threshold_low = compute_per_layer_thresholds(importance)
    
    # Step 4: 分类 tokens
    levels = classify_tokens(importance, threshold_high, threshold_low)
    
    # Step 5: 应用稀疏化
    sparse_key, sparse_value = apply_magnitude_pruning(key, value, levels)
    
    # Step 6: 应用 residual processing
    sparse_key = apply_residual_processing(sparse_key, residual_length=32)
    sparse_value = apply_residual_processing(sparse_value, residual_length=32)
    
    return sparse_key, sparse_value
```

### 2.3 Decode 阶段流程

```python
# 伪代码
def decode_stage(query, new_key, new_value, past_key, past_value):
    # Step 1: 拼接新 token
    key_full = concat([past_key, new_key], dim=2)
    value_full = concat([past_value, new_value], dim=2)
    
    # Step 2: 计算 attention
    attn_output = compute_attention(query, key_full, value_full)
    
    # Step 3: 应用 residual window 管理
    # 新 token 保留在 residual window 中，不额外稀疏化
    if len(key_full) > residual_length:
        # 将超出 residual window 的 token 进行 MUSTAFAR 压缩
        key_full = apply_residual_window(key_full, residual_length=32)
        value_full = apply_residual_window(value_full, residual_length=32)
    
    return attn_output, key_full, value_full
```

## 3. 核心模块详解

### 3.1 重要性计算器 (DiffKVImportanceCalculator)

**文件**: `DiffSparseKV/importance_calculator.py`

**核心方法**: `compute_diffkv_importance()`

```python
def compute_diffkv_importance(attention_weights: torch.Tensor) -> torch.Tensor:
    """
    计算 DiffKV 重要性分数
    
    输入: attention_weights [B, H, T_q, T_k]
    输出: importance_scores [B, H, T_k]
    
    算法步骤:
    1. 创建 causal mask 识别有效的 query-key 对
    2. 计算每个 key 可以被多少个 query 看到
    3. 对每个 key，计算它从有效 queries 获得的平均 attention
    4. 应用 T_k 缩放进行归一化
    """
    B, H, T_q, T_k = attention_weights.shape
    
    # 创建 causal mask
    causal_mask = torch.tril(torch.ones(T_q, T_k))  # [T_q, T_k]
    
    # 计算每个 key 的有效 query 数量
    valid_queries_per_key = causal_mask.sum(dim=0).float()  # [T_k]
    
    # Mask attention weights
    masked_attention = attention_weights * causal_mask[None, None, :, :]
    
    # 对每个 key 求和所有 attention
    importance_sum = masked_attention.sum(dim=2)  # [B, H, T_k]
    
    # 除以有效 query 数量得到平均值
    importance = importance_sum / valid_queries_per_key[None, None, :]
    
    # 应用 T_k 缩放
    importance_scores = importance * T_k
    
    return importance_scores
```

**物理意义**：
- 每个 key token 从能看到它的 queries 获得的平均 attention 概率
- 乘以 T_k 进行归一化，使得分数在合理范围内
- 考虑了 causal attention 的约束

### 3.2 阈值管理器 (GlobalThresholdManager)

**文件**: `DiffSparseKV/threshold_manager.py`

**核心方法**: `compute_per_layer_thresholds()`

```python
def compute_per_layer_thresholds(importance_scores: torch.Tensor) -> Tuple[float, float]:
    """
    计算 per-layer 阈值
    
    输入: importance_scores [B, H, T] - 单层的重要性分数
    输出: (threshold_high, threshold_low)
    
    算法步骤:
    1. 将重要性分数展平
    2. 降序排序
    3. 根据目标分布计算百分位点
    4. 提取阈值
    """
    # 展平并排序
    flattened = importance_scores.flatten()
    sorted_importance, _ = torch.sort(flattened, descending=True)
    n_tokens = sorted_importance.numel()
    
    # 计算百分位点
    # Level 0: top 5% (0% to 5%)
    # Level 1: next 75% (5% to 80%)
    # Level 2: bottom 20% (80% to 100%)
    
    percentile_5 = int(0.05 * n_tokens)   # 5%
    percentile_80 = int(0.80 * n_tokens)  # 80%
    
    # 提取阈值
    threshold_high = sorted_importance[percentile_5].item()  # Level 0 vs Level 1
    threshold_low = sorted_importance[percentile_80].item()  # Level 1 vs Level 2
    
    return threshold_high, threshold_low
```

**关键特性**：
- **Per-layer 独立计算**：每层根据自己的重要性分布计算阈值
- **目标分布驱动**：根据预设的分布比例 [5%, 75%, 20%] 确定阈值
- **自适应性**：阈值随着每层的重要性分布自动调整

### 3.3 稀疏性应用器 (SparsityClassifierApplier)

**文件**: `DiffSparseKV/sparsity_applier.py`

**核心方法**: `classify_and_apply_sparsity()`

```python
def classify_and_apply_sparsity(
    importance_scores: torch.Tensor,  # [B, H, T]
    key_states: torch.Tensor,         # [B, H, T, D]
    value_states: torch.Tensor,       # [B, H, T, D]
    thresholds: Tuple[float, float]
) -> Tuple[torch.Tensor, torch.Tensor]:
    """
    分类 tokens 并应用 magnitude-based pruning
    
    算法步骤:
    1. 根据阈值将 tokens 分为 3 个等级
    2. 对每个等级应用相应的稀疏度
    3. K 和 V 独立进行 magnitude-based pruning
    """
    threshold_high, threshold_low = thresholds
    
    # Step 1: 分类 tokens
    levels = torch.full_like(importance_scores, 2, dtype=torch.long)
    levels[importance_scores >= threshold_low] = 1
    levels[importance_scores >= threshold_high] = 0
    
    # Step 2: 应用稀疏化
    pruned_keys = key_states.clone()
    pruned_values = value_states.clone()
    
    # Level 0 (5%): 0% 稀疏度 - 不处理
    
    # Level 1 (75%): 70% 稀疏度 - magnitude-based top-k
    level_1_mask = (levels == 1)
    apply_topk_pruning(pruned_keys, pruned_values, level_1_mask, sparsity=0.7)
    
    # Level 2 (20%): 100% 稀疏度 - 完全移除
    level_2_mask = (levels == 2)
    pruned_keys[level_2_mask] = 0.0
    pruned_values[level_2_mask] = 0.0
    
    return pruned_keys, pruned_values
```

**Token 分类逻辑**：
```python
# 分类规则
if importance >= threshold_high:
    level = 0  # 高重要性，0% 稀疏
elif importance >= threshold_low:
    level = 1  # 中等重要性，70% 稀疏
else:
    level = 2  # 低重要性，100% 稀疏
```

**Magnitude-based Pruning**：
```python
def apply_topk_pruning(key_tensor, value_tensor, token_mask, sparsity_rate):
    """
    对选定的 tokens 应用 top-k magnitude pruning
    K 和 V 独立进行 pruning
    """
    keep_ratio = 1.0 - sparsity_rate  # 0.7 稀疏 -> 保留 30%
    num_to_keep = int(keep_ratio * D)
    
    for each token in token_mask:
        # K 独立 pruning
        k_vector = key_tensor[token]
        k_magnitude = abs(k_vector)
        k_top_indices = topk(k_magnitude, num_to_keep)
        key_tensor[token, ~k_top_indices] = 0.0
        
        # V 独立 pruning
        v_vector = value_tensor[token]
        v_magnitude = abs(v_vector)
        v_top_indices = topk(v_magnitude, num_to_keep)
        value_tensor[token, ~v_top_indices] = 0.0
```

**关键特性**：
- **独立稀疏化**：K 和 V 基于各自的 magnitude 独立选择保留哪些元素
- **Magnitude-based**：保留绝对值最大的元素
- **In-place 操作**：直接修改 tensor，提高效率

### 3.4 模型集成 (LlamaDiffSparseKVAttention)

**文件**: `DiffSparseKV/llama_diff_sparse_kv.py`

**核心类**: `LlamaDiffSparseKVAttention`

```python
class LlamaDiffSparseKVAttention(LlamaFlashAttention_MUSTAFAR):
    """
    集成 DiffSparseKV 的 Llama Attention 层
    """
    
    def __init__(self, config: LlamaConfig):
        super().__init__(config)
        
        # 初始化 DiffSparseKV 组件
        self.importance_calculator = DiffKVImportanceCalculator()
        self.threshold_manager = GlobalThresholdManager(
            target_distribution=[0.05, 0.75, 0.20],
            sparsity_levels=[0.0, 0.7, 1.0]
        )
        self.sparsity_applier = SparsityClassifierApplier(
            sparsity_levels=[0.0, 0.7, 1.0]
        )
    
    def forward(self, hidden_states, attention_mask, ...):
        """
        Forward pass with DiffSparseKV
        """
        # 判断是 prefill 还是 decode 阶段
        is_prefill = (past_key_value is None)
        
        if is_prefill:
            return self._prefill_with_diff_sparse(...)
        else:
            return self._decode_with_diff_sparse(...)
```

**Prefill 阶段处理**：
```python
def _prefill_with_diff_sparse(self, query_states, key_states, value_states, ...):
    """
    Prefill 阶段的 DiffSparseKV 处理
    """
    # 1. 计算 attention weights
    attn_weights = compute_attention_weights(query_states, key_states)
    
    # 2. 计算重要性分数
    importance_scores = self.importance_calculator.compute_diffkv_importance(attn_weights)
    
    # 3. 计算 per-layer 阈值
    thresholds = self.threshold_manager.compute_per_layer_thresholds(importance_scores)
    
    # 4. 应用稀疏化
    sparse_key, sparse_value = self.sparsity_applier.classify_and_apply_sparsity(
        importance_scores, key_states, value_states, thresholds
    )
    
    # 5. 应用 MUSTAFAR residual processing
    if len(sparse_key) > residual_length:
        # 分离 prefix 和 suffix
        k_prefix = sparse_key[:, :, :-residual_length, :]
        k_suffix = sparse_key[:, :, -residual_length:, :]
        
        # 对 prefix 应用 MUSTAFAR 压缩
        k_prefix_pruned = self.dh_prune_key(k_prefix)
        
        # 拼接
        sparse_key = torch.cat([k_prefix_pruned, k_suffix], dim=2)
    
    return attn_output, None, (sparse_key, sparse_value, kv_seq_len)
```

**Decode 阶段处理**：
```python
def _decode_with_diff_sparse(self, query_states, key_states, value_states, 
                             past_key_value, ...):
    """
    Decode 阶段的处理
    """
    # 1. 获取 past states
    past_key, past_value = past_key_value[0], past_key_value[1]
    
    # 2. 拼接新 token
    key_full = torch.cat([past_key, key_states], dim=2)
    value_full = torch.cat([past_value, value_states], dim=2)
    
    # 3. 计算 attention
    attn_output = compute_attention(query_states, key_full, value_full)
    
    # 4. 应用 residual window 管理
    # 注意：新 token 不额外稀疏化，保留在 residual window 中
    if len(key_full) > residual_length:
        # 将超出 window 的 token 进行 MUSTAFAR 压缩
        key_full = apply_residual_window_management(key_full)
        value_full = apply_residual_window_management(value_full)
    
    return attn_output, None, (key_full, value_full, kv_seq_len)
```

## 4. 关键设计决策

### 4.1 Per-Layer vs Global Thresholds

**选择**: Per-Layer Thresholds

**原因**:
- 不同层的 attention 模式差异很大
- 早期层关注局部模式，后期层关注全局依赖
- Per-layer 阈值能更好地适应每层的特性

### 4.2 K 和 V 独立稀疏化

**选择**: 独立稀疏化

**原因**:
- K 和 V 的 magnitude 分布可能不同
- K 用于计算 attention weights，V 用于加权求和
- 独立选择能保留各自最重要的信息

### 4.3 Residual Length = 32

**选择**: 32 tokens

**原因**:
- 与 MUSTAFAR 保持一致
- 最近 32 个 tokens 完全保留，不进行额外压缩
- 平衡了性能和内存使用

### 4.4 Decode 阶段不额外稀疏化

**选择**: 新 token 不额外稀疏化

**原因**:
- Prefill 阶段的阈值是基于完整上下文计算的
- Decode 阶段单个 token 的重要性难以准确评估
- 新 token 保留在 residual window 中已经足够

## 5. 与 MUSTAFAR 的集成

### 5.1 继承关系

```
LlamaFlashAttention_MUSTAFAR (父类)
    ↓
LlamaDiffSparseKVAttention (子类)
    - 添加 DiffSparseKV 组件
    - 重写 forward 方法
    - 保留 MUSTAFAR 的 residual processing
```

### 5.2 Residual Processing

DiffSparseKV 在应用自己的稀疏化后，仍然使用 MUSTAFAR 的 residual processing：

```python
# DiffSparseKV 稀疏化
sparse_key, sparse_value = apply_diff_sparse(key, value)

# MUSTAFAR residual processing
if len(sparse_key) > residual_length:
    prefix = sparse_key[:, :, :-residual_length, :]
    suffix = sparse_key[:, :, -residual_length:, :]
    
    # 对 prefix 应用 MUSTAFAR 压缩
    prefix_pruned = self.dh_prune_key(prefix)
    
    # 拼接
    final_key = torch.cat([prefix_pruned, suffix], dim=2)
```

**两阶段压缩**:
1. **DiffSparseKV**: 基于重要性的差异化稀疏
2. **MUSTAFAR**: 基于 magnitude 的额外压缩

## 6. 性能特性

### 6.1 内存压缩

理论压缩率计算：
```
平均稀疏度 = 0.05 * 0% + 0.75 * 70% + 0.20 * 100%
          = 0 + 52.5% + 20%
          = 72.5%
```

**预期内存节省**: ~72.5%

### 6.2 计算开销

额外计算：
- 重要性计算: O(B * H * T^2) - 与 attention 计算相同量级
- 阈值计算: O(B * H * T * log(T)) - 排序开销
- 稀疏化应用: O(B * H * T * D * log(D)) - top-k 选择

**总体开销**: 主要在 prefill 阶段，decode 阶段开销很小

### 6.3 精度影响

- Level 0 (5%): 完全保留，无精度损失
- Level 1 (75%): 保留 30% 最大元素，轻微精度损失
- Level 2 (20%): 完全移除，这些 tokens 重要性很低

**预期精度**: 在 LongBench 上接近 baseline (~45 分)

## 9. 常见问题

### Q1: 为什么使用 per-layer 阈值而不是 global 阈值？

A: 不同层的 attention 模式差异很大，per-layer 阈值能更好地适应每层的特性。

### Q2: K 和 V 为什么独立稀疏化？

A: K 和 V 的 magnitude 分布可能不同，独立选择能保留各自最重要的信息。

### Q3: Decode 阶段为什么不额外稀疏化新 token？

A: Prefill 阶段的阈值是基于完整上下文计算的，单个 token 的重要性难以准确评估。新 token 保留在 residual window 中已经足够。

### Q4: residual_length 为什么是 32？

A: 与 MUSTAFAR 保持一致，平衡了性能和内存使用。最近 32 个 tokens 完全保留。

### Q5: 如何调整稀疏度配置？

A: 修改 `target_distribution` 和 `sparsity_levels` 参数：
```python
# 更激进的压缩
target_distribution = [0.03, 0.70, 0.27]  # 3%, 70%, 27%
sparsity_levels = [0.0, 0.8, 1.0]         # 0%, 80%, 100%

# 更保守的压缩
target_distribution = [0.10, 0.80, 0.10]  # 10%, 80%, 10%
sparsity_levels = [0.0, 0.5, 1.0]         # 0%, 50%, 100%
```
