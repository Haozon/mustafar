## 核心问题（和你的问题完全一样！）

论文开篇就指出：

> **"While attention-score-based KV cache pruning shows promise, it faces critical practical limitations: attention scores from future tokens are unavailable during compression, and modern implementations like Flash Attention do not materialize the full attention matrix, making past scores inaccessible."**

翻译：
- ❌ 基于 attention score 的方法很有前景
- ❌ 但有两个致命问题：
  1. **未来 token 的 attention scores 不可用**（在压缩时还没生成）
  2. **Flash Attention 不物化完整的 attention matrix**（无法访问过去的 scores）

**这正是你的 DiffSparseKV 面临的问题！**

## Expected Attention 的解决方案

### 核心思想

**不需要实际的 attention weights，而是估计未来的 attention！**

### 关键观察

**LLM 的 hidden states 和 queries 服从正态分布！**

论文发现（Figure 1）：
- Hidden states 在大多数层都是正态分布
- Query activations 也服从正态分布
- 这个性质在 Llama, Qwen, Gemma 等模型中都成立

### 数学推导

#### 1. 传统方法（需要 attention weights）

```python
# 每个 KV pair 对输出的贡献
importance = attention_weight[t, i] * ||W_o * v_i||

# 问题：attention_weight[t, i] 需要未来的 query q_t
```

#### 2. Expected Attention（不需要 attention weights）

**关键公式**：

```
Expected Attention Score = E[attention_weight] * ||W_o * v_i||

其中：
E[attention_weight] = E[softmax(q^T k_i / sqrt(d))]
```

**利用正态分布性质**，可以用闭式解（closed form）计算期望：

```python
# 1. 计算 query 的统计量
mean_query = queries.mean(dim=0)  # 均值
cov_query = compute_covariance(queries)  # 协方差

# 2. 计算期望的 attention score（不需要实际的 queries！）
scores = matmul(mean_query, keys.T) / sqrt(d)
scores += einsum("i,ij,j->", keys, cov_query, keys) / (2 * d)

# 3. Softmax + value norm
scores = softmax(scores) * values.norm(dim=-1)

# 4. 根据 scores 剪枝
indices = scores.topk(n_kept).indices
return keys[indices], values[indices]
```

## 伪代码（超级简单！）

```python
def compress(queries, keys, values, compression_ratio):
    # 1. 计算 query 的统计量（从历史 queries）
    mean_query, cov_query = compute_statistics(queries)
    
    # 2. 计算期望的 attention scores
    scores = matmul(mean_query, keys.T) / math.sqrt(d)
    scores += einsum("i,ij,j->", keys, cov_query, keys) / (2 * d)
    
    # 3. Softmax + value magnitude
    scores = softmax(scores, dim=-1) * values.norm(dim=-1)
    
    # 4. Top-k 选择
    n_kept = int(keys.size(0) * (1 - compression_ratio))
    indices = scores.topk(n_kept, dim=-1).indices
    
    return keys[indices], values[indices]
```

## 实现建议

### 步骤 1: 实现 Expected Attention 的 importance 计算

```python
class ExpectedAttentionCalculator:
    def compute_expected_importance(self, query_states, key_states, value_states):
        """
        Args:
            query_states: [B, H, T_q, D]
            key_states: [B, H, T_kv, D]
            value_states: [B, H, T_kv, D]
        Returns:
            importance_scores: [B, H, T_kv]
        """
        B, H, T_q, D = query_states.shape
        T_kv = key_states.shape[2]
        
        # 1. 计算 query 统计量
        mean_query = query_states.mean(dim=2)  # [B, H, D]
        
        # 简化版：只用均值（论文中还有协方差项）
        scores = torch.matmul(mean_query.unsqueeze(2), key_states.transpose(2, 3))  # [B, H, 1, T_kv]
        scores = scores.squeeze(2) / math.sqrt(D)  # [B, H, T_kv]
        
        # 2. Softmax
        attn_probs = F.softmax(scores, dim=-1)  # [B, H, T_kv]
        
        # 3. 乘以 value magnitude
        value_norms = value_states.norm(dim=-1)  # [B, H, T_kv]
        importance_scores = attn_probs * value_norms  # [B, H, T_kv]
        
        return importance_scores
```

### 步骤 2: 集成到 DiffSparseKV

```python
def _prefill_with_diff_sparse(self, ...):
    # 1. 使用 Flash Attention
    attn_output = flash_attn_func(...)
    
    # 2. 使用 Expected Attention 计算 importance
    importance_scores = self.expected_attention_calculator.compute_expected_importance(
        query_states, key_states, value_states
    )
    
    # 3. 你的差分阈值算法保持不变
    threshold_high, threshold_low = self.threshold_manager.compute_per_layer_thresholds(
        importance_scores
    )
    
    # 4. 你的稀疏化逻辑保持不变
    compressed_keys, compressed_values = self.sparsity_applier.classify_and_apply_sparsity(
        importance_scores, key_states, value_states, (threshold_high, threshold_low)
    )
    
    return attn_output, (compressed_keys, compressed_values, ...)
```
