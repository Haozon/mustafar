# DiffSparseKV 阈值机制详解

## 📋 问题描述

在 Decode 阶段，每个 token 的稀疏化需要使用阈值来分类（Level 0/1/2）。这些阈值应该来自 Prefill 阶段，以保持一致性。

## 🎯 阈值的作用

### 分类规则

```python
# 对于每个 token 的重要性分数 importance_score:
if importance_score >= threshold_high:
    level = 0  # 保留全部（0% 稀疏）
elif importance_score >= threshold_low:
    level = 1  # 保留 30%（70% 稀疏）
else:
    level = 2  # 完全删除（100% 稀疏）
```

### 阈值的含义

- **threshold_high**: Level 0 和 Level 1 的边界（前 5% 的边界）
- **threshold_low**: Level 1 和 Level 2 的边界（前 80% 的边界）

## 🔄 阈值的生命周期

### 阶段 1: Prefill 阶段计算阈值

```python
def _prefill_with_diff_sparse(self, ...):
    # 1. 计算 attention weights（不使用 Flash Attention）
    attn_weights = compute_attention(...)
    
    # 2. 计算 importance scores
    importance_scores = self.importance_calculator.compute_diffkv_importance(
        attn_weights  # [B, H, T_q, T_k]
    )  # 返回 [B, H, T_k]
    
    # 3. 计算 per-layer thresholds
    threshold_high, threshold_low = self.threshold_manager.compute_per_layer_thresholds(
        importance_scores  # [B, H, T]
    )
    
    # 4. 保存 thresholds 到实例变量（关键！）
    self.diff_sparse_thresholds = (threshold_high, threshold_low)
    self.thresholds_computed = True
    
    # 5. 应用稀疏化
    sparse_keys, sparse_values = self.sparsity_applier.classify_and_apply_sparsity(
        importance_scores, key_states, value_states, 
        thresholds=(threshold_high, threshold_low)
    )
    
    # 6. 初始化双窗口
    self.initialize_dual_window_after_prefill(sparse_keys, sparse_values)
```

### 阶段 2: Decode 阶段复用阈值

```python
def _decode_with_diff_sparse(self, ...):
    # 1. 检查 thresholds 是否已计算
    if not self.thresholds_computed or self.diff_sparse_thresholds is None:
        raise RuntimeError("Thresholds not computed. Must run prefill first.")
    
    # 2. 获取保存的 thresholds
    threshold_high, threshold_low = self.diff_sparse_thresholds
    
    # 3. 在压缩 Window A 时使用这些 thresholds
    if self.window_state['window_a_size'] == W and self.window_state['window_b_size'] >= W:
        # 归一化 attention
        normalized_attention = normalize_accumulated_attention(...)
        
        # 使用 Prefill 的 thresholds 分类
        sparse_keys, sparse_values = self.sparsity_applier.classify_and_apply_sparsity(
            normalized_attention, 
            window_a_keys, 
            window_a_values,
            thresholds=(threshold_high, threshold_low)  # 复用 Prefill 的阈值
        )
```

## 🔧 实现方案

### 使用实例变量（推荐）

**优点**：
- 简单直接
- 不需要修改 past_key_value 格式
- 每个 attention layer 独立管理自己的 thresholds
- 与 window_state 的存储方式一致

**实现**：

```python

                "Thresholds not computed. Must run prefill stage first before decode."

        # 2. 获取保存的 thresholds
        threshold_high, threshold_low = self.diff_sparse_thresholds

        attn_output = self.decode_step_with_dual_window(

        # （已在 decode_step_with_dual_window 中自动检查和触发）

        # ... 继续处理 ...
```

## 📝 Per-Layer vs Global Thresholds

### 当前实现：Per-Layer Thresholds

```python
# 每个 layer 独立计算 thresholds
class LlamaModel_DiffSparseKV:
    def __init__(self, config):
        self.layers = nn.ModuleList([
            LlamaDiffSparseKVDecoderLayer(config) 
            for _ in range(num_layers)
        ])
        
        # 每个 layer 的 attention 模块有自己的 thresholds
        # self.layers[0].self_attn.diff_sparse_thresholds
        # self.layers[1].self_attn.diff_sparse_thresholds
        # ...
```

**优点**：
- 每层自适应，适应不同层的注意力模式
- 浅层和深层可以有不同的稀疏策略

**缺点**：
- 不同层的阈值不同，可能导致不一致
- 需要为每层单独存储

### 替代方案：Global Thresholds

```python
# 所有层共享一个全局 threshold
class LlamaModel_DiffSparseKV:
    def __init__(self, config):
        # 全局 threshold manager
        self.global_threshold_manager = GlobalThresholdManager(...)
        self.global_thresholds = None
        
    def forward(self, ...):
        # 在第一层 prefill 时计算全局 thresholds
        if self.global_thresholds is None:
            # 收集所有层的 importance scores
            all_importance = []
            for layer in self.layers:
                importance = layer.compute_importance(...)
                all_importance.append(importance)
            
            # 计算全局 thresholds
            self.global_thresholds = self.global_threshold_manager.compute_global_thresholds(
                torch.cat(all_importance)
            )
        
        # 所有层使用相同的 thresholds
        for layer in self.layers:
            layer.self_attn.diff_sparse_thresholds = self.global_thresholds
```

**优点**：
- 所有层一致
- 只需存储一次

**缺点**：
- 不能适应不同层的特性
- 需要收集所有层的 importance scores（复杂）

## 🔍 论文中的方案

根据你提供的论文（Section 4.2.2）：

> "需要注意的是，我们在Decode阶段**不重新计算阈值**，而是复用Prefill阶段的阈值。这基于一个合理的假设：同一层内，token重要性的相对分布在Prefill和Decode阶段保持相对稳定。"

**论文使用的是 Per-Layer Thresholds**：
- 每层在 Prefill 阶段计算自己的 thresholds
- Decode 阶段复用该层的 thresholds
- 不同层可以有不同的 thresholds

## 🎯 总结

1. **阈值来源**：Prefill 阶段计算，Decode 阶段复用
2. **存储方式**：使用实例变量 `self.diff_sparse_thresholds`
3. **计算方法**：Per-layer thresholds（每层独立）
4. **使用时机**：Decode 阶段压缩 Window A 时使用
