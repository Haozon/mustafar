## 🎯 核心功能实现

### 1. 双窗口机制 ✅

```python
# Window State 结构
{
    'compressed_keys': [B, H_kv, N_compressed, D],
    'compressed_values': [B, H_kv, N_compressed, D],
    'window_a_keys': [B, H_kv, <=W, D],
    'window_a_values': [B, H_kv, <=W, D],
    'window_a_size': int,
    'window_b_keys': [B, H_kv, <=W, D],
    'window_b_values': [B, H_kv, <=W, D],
    'window_b_size': int,
    'accumulator': [B, H_kv, W],
    'token_observation_count': [B, H_kv, W],
}
```

**实现要点**:
- ✅ Window A: 可压缩窗口
- ✅ Window B: 保护窗口
- ✅ 动态大小支持 (window_a_size, window_b_size)
- ✅ 实例变量存储

### 2. Attention 累积 ✅

```python
# 每个 decode step
if window_a_size > 0:
    window_a_attention = attn_weights_kv[:, :, 0, window_a_start:window_a_end]
    accumulator[:, :, :window_a_size] += window_a_attention
    token_observation_count[:, :, :window_a_size] += 1
```

**实现要点**:
- ✅ 动态累积 (无论 Window A 是否满)
- ✅ token_observation_count 跟踪每个 token 的观察次数
- ✅ 支持 Window A 填充期间的累积

### 3. 动态归一化 ✅

```python
for i in range(W):
    N_internal = W - 1 - i
    N_external = token_observation_count[:, :, i]
    N_total = N_internal + N_external
    N_total = torch.clamp(N_total, min=1.0)
    ᾱ[:, :, i] = Acc[:, :, i] / N_total
```

**实现要点**:
- ✅ 动态计算观察次数
- ✅ 考虑 Window A 填充期间的差异
- ✅ 避免除零错误

### 4. GQA 支持 ✅

**实现要点**:
- ✅ GQA 配置检查

- ✅ Attention 聚合 (mean)
- ✅ Accumulator 维度正确 ([B, H_kv, W])

### 5. Prefill 处理 ✅

**Case A: Prefill >= W**
```python
# 稀疏化前 (N-W) tokens
compressed_keys, compressed_values = sparsify(prefix)
# 最后 W tokens 保持 dense
window_keys = key_states[:, :, -W:, :]
```

**Case B: Prefill < W**
```python
# 全部作为 Window A，不稀疏化
window_a_keys = key_states
window_a_size = N  # < W
```

**实现要点**:
- ✅ 分情况处理
- ✅ Thresholds 计算并保存
- ✅ 双窗口初始化

### 6. Decode 处理 ✅

```python
if window_a_size < W:
    # 添加到 Window A (填充)
    window_a.append(new_token)
else:
    # 添加到 Window B
    window_b.append(new_token)

# 累积 attention
accumulator += window_a_attention
token_observation_count += 1

# 检查压缩
if window_a_size == W and window_b_size >= W:
    trigger_compression()
```

**实现要点**:
- ✅ 动态添加到 Window A 或 Window B
- ✅ 持续累积 attention
- ✅ 压缩条件检查
