# 🎓 Expected Attention 简单解释

## 问题回顾

### 你的 DiffSparseKV 为什么慢？

```python
# 你的当前实现
def prefill(Q, K, V):
    # 步骤 1: 计算完整的 attention matrix
    attn_weights = Q @ K.T  # [B, 32, 2048, 2048] = 4.3 GB！
    attn_weights = softmax(attn_weights)
    
    # 步骤 2: 计算 importance（需要 attn_weights）
    importance = attn_weights.sum(dim=-2)  # 每个 token 被 attend 的总量
    
    # 步骤 3: 根据 importance 做稀疏化
    ...
    
    # 问题：步骤 1 太慢了！（18 秒）
```

**为什么不能用 Flash Attention？**
- Flash Attention 不返回 `attn_weights`
- 但你需要 `attn_weights` 来计算 `importance`

---

## Expected Attention 的核心思想

### 关键洞察

**不需要计算实际的 attention weights，可以"估计"它们！**

### 类比理解

想象你要预测一个班级的考试成绩分布：

**方法 1（你的方法）**：
- 等所有学生考完试
- 收集所有成绩
- 计算平均分、分布等

**方法 2（Expected Attention）**：
- 不等考试
- 根据学生的历史成绩（均值、方差）
- **估计**这次考试的成绩分布

**关键**：如果学生成绩服从正态分布，可以用统计方法估计！

---

## 数学原理（简化版）

### 你需要什么？

```python
# 你需要计算每个 token 的 importance
importance[i] = sum(attention_weights[:, i])

# attention_weights[t, i] = softmax(Q[t] @ K[i] / sqrt(d))
```

**问题**：需要所有的 Q[t]（包括未来的 tokens）

### Expected Attention 的解决方案

**观察**：LLM 的 Query 向量服从正态分布！

```
Q ~ Normal(μ, Σ)
其中：
- μ = 均值向量
- Σ = 协方差矩阵
```

**推导**：

```python
# 传统方法（需要所有 Q）
importance[i] = sum over t: softmax(Q[t] @ K[i] / sqrt(d))

# Expected Attention（只需要 Q 的统计量）
expected_importance[i] = E[softmax(Q @ K[i] / sqrt(d))]

# 利用正态分布性质，可以用闭式解计算期望：
expected_importance[i] ≈ softmax(μ @ K[i] / sqrt(d) + correction_term)

其中 correction_term 来自协方差矩阵
```

## 具体实现（超级简单）

### 步骤 1: 收集 Query 的统计量

```python
# 从已有的 queries 计算统计量
def compute_query_statistics(queries):
    """
    Args:
        queries: [B, H, T, D] - 已经生成的 queries
    Returns:
        mean_query: [B, H, D] - 均值
        cov_query: [B, H, D, D] - 协方差（可选）
    """
    mean_query = queries.mean(dim=2)  # 对 T 维度求平均
    
    # 简化版：只用均值（已经很好了）
    return mean_query
```

**关键**：
- Prefill 阶段：用输入 prompt 的 queries
- Decode 阶段：用最近生成的 queries（比如最近 128 个）

### 步骤 2: 计算 Expected Importance

```python
def compute_expected_importance(mean_query, keys, values):
    """
    Args:
        mean_query: [B, H, D] - query 的均值
        keys: [B, H, T, D] - 所有的 keys
        values: [B, H, T, D] - 所有的 values
    Returns:
        importance: [B, H, T] - 每个 token 的重要性
    """
    B, H, T, D = keys.shape
    
    # 1. 计算 "平均 query" 和每个 key 的相似度
    scores = torch.matmul(
        mean_query.unsqueeze(2),  # [B, H, 1, D]
        keys.transpose(2, 3)       # [B, H, D, T]
    ).squeeze(2) / math.sqrt(D)    # [B, H, T]
    
    # 2. Softmax（得到 "期望的 attention weights"）
    expected_attn = F.softmax(scores, dim=-1)  # [B, H, T]
    
    # 3. 乘以 value 的 magnitude
    value_norms = values.norm(dim=-1)  # [B, H, T]
    importance = expected_attn * value_norms  # [B, H, T]
    
    return importance
```

**就这么简单！**

## 为什么这样可行？

### 理论基础

1. **Query 服从正态分布**
   - 论文通过实验验证了这一点
   - 在 Llama, Qwen, Gemma 等模型中都成立

2. **期望可以用闭式解计算**
   - 如果 Q ~ Normal(μ, Σ)
   - 那么 E[softmax(Q @ K)] 可以近似计算
   - 主要项就是 softmax(μ @ K)

3. **近似足够准确**
   - 论文实验显示，Expected Attention 和真实 Attention 高度相关
   - 在实际任务中性能几乎无损

## 为什么叫 "Expected" Attention？

**Expected = 期望值（统计学概念）**

```
传统方法：
importance[i] = sum(attention[t, i])  # 对所有 t 求和

Expected Attention:
importance[i] = E[attention[t, i]]    # 计算期望值
              = sum(P(Q=q) * attention(q, i))  # 对所有可能的 q 加权求和
              ≈ attention(E[Q], i)              # 近似为用均值计算
```

## 总结

### 核心思想（一句话）

**用 Query 的统计量（均值）代替实际的 Queries，估计 Attention 的期望值。**

### 为什么可行？

1. Query 服从正态分布（实验验证）
2. 期望值可以用均值近似（数学推导）
3. 近似足够准确（实验验证）

### 你需要做什么？

1. 把 `attn_weights = Q @ K.T` 改成 `mean_query @ K.T`
2. 其他代码几乎不变
3. 可以使用 Flash Attention（快 180x）

### 最大的好处

- ✅ 保留你的算法创新（差分阈值、双窗口）
- ✅ 获得 Flash Attention 的速度
- ✅ 实现简单（改动很小）
- ✅ 有论文支持（NVIDIA 2025年1月发表）
